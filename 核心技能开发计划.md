## 一、优先级评估框架

### 1.1 评估维度（每项 1-5 分）

| 维度 | 说明 | 权重 |
|------|------|------|
| **用户需求强度** | PM 日常工作中使用频率（高=5，低=1） | 30% |
| **商业价值** | 是否值得付费（高=5，低=1） | 25% |
| **开发复杂度** | 云端化适配难度（简单=5，复杂=1） | 20% |
| **差异化优势** | 与竞品相比的独特性（强=5，弱=1） | 15% |
| **本土化潜力** | 中国场景适配深度（高=5，低=1） | 10% |

**综合得分** = Σ(维度得分 × 权重)

### 1.2 优先级分类

- **P0（立即开发）**: 综合得分 ≥ 4.0，且用户需求强度 ≥ 4
- **P1（短期开发）**: 综合得分 3.5-4.0，或用户需求强度 = 5
- **P2（中期开发）**: 综合得分 3.0-3.5
- **P3（长期开发）**: 综合得分 < 3.0

## 四、关键任务详细说明

### 4.1 云端化适配（技术核心）

**问题**：当前技能设计为本地运行（可读写文件），但扣子平台是云端运行，无法读写本地文件。

**解决方案**：

#### 方案 A：输出改为对话中展示（推荐）
- **适用场景**：文本内容（如 PRD、用户故事、问题陈述）
- **实现方式**：
  - 技能输出 Markdown 文本，直接在扣子对话中展示
  - 用户提供"复制到剪贴板"按钮（扣子平台支持）
  - 用户手动粘贴到本地文件（如腾讯文档、飞书文档）
  
- **示例**：

### 5.2 免费+付费混合模式

#### 免费技能（引流款）
- **目的**：降低用户尝试门槛，建立信任，引流到付费技能
- **选择标准**：
  - 使用频率高（如 user-story-cn, problem-statement-cn）
  - 能展示技能质量（用户试用后愿意购买其他技能）
  - 不易盗版（如引导式 Interactive 技能，依赖 AI 对话）
  
- **推荐免费技能（5-10 个）**：
  - `user-story-cn` - 用户故事（最常用）
  - `problem-statement-cn` - 问题陈述（入门必备）
  - `jobs-to-be-done-cn` - JTBD 分析（基础框架）
  - `proto-persona-cn` - 用户画像（基础技能）
  - `positioning-statement-cn` - 定位陈述（经典框架）

#### 付费技能（利润款）
- **目的**：实现变现，覆盖开发和维护成本
- **选择标准**：
  - 商业价值高（如 prd-development-cn, product-strategy-session-cn）
  - 输出复杂（如 Workflow 技能，生成完整文档）
  - 差异化强（如 executive-onboarding-playbook-cn, ai-shaped-readiness-advisor-cn）
  
- **推荐付费技能（15-20 个）**：
  - 所有 Workflow 类型技能（8 个）
  - 高价值的 Interactive 类型技能（7 个）
  - 部分高级 Component 类型技能（5 个）

### 5.3 订阅模式 vs 一次性购买

#### 方案 A：一次性购买（推荐）
- **优势**：
  - 用户心理门槛低（"只付一次，永久使用"）
  - 适合中国用户习惯（偏好买断制，而非订阅制）
  
- **劣势**：
  - 无法持续产生收入
  - 需要不断开发新技能或获取新用户
  
- **适用场景**：
  - 技能更新频率低（如 PRD 模板、用户故事模板）
  - 用户一次性需求（如面试准备、EOL 公告）

#### 方案 B：订阅模式（备选）
- **优势**：
  - 持续产生收入（适合 SaaS 模式）
  - 可以持续更新技能，提供新功能
  
- **劣势**：
  - 用户心理门槛高（"每个月都要付钱"）
  - 需要持续提供价值，否则用户会取消订阅
  
- **适用场景**：
  - 技能需要持续更新（如市场数据、竞品分析）
  - 提供持续服务（如一对一咨询、团队培训）

**推荐路径**：先采用一次性购买模式（快速获取第一批付费用户），再根据用户反馈决定是否推出订阅模式（如"Professional Pack 订阅版：¥19.9/月，包含所有付费技能的持续更新"）。

## 六、风险与应对

### 6.1 技术风险

**风险 1**：扣子平台的 Skill 开发环境与本地不同，可能出现兼容性问题

**应对措施**：
- 在扣子平台逐一测试所有技能
- 准备备用方案：如果扣子平台无法实现某些功能，可以提供"扣子平台版本"和"本地版本"两个版本
- 与扣子平台技术支持保持沟通，及时反馈问题

**风险 2**：云端化适配可能导致技能质量下降（如无法生成复杂可视化）

**应对措施**：
- 优先适配文本内容输出（不易受云端环境影响的技能）
- 对于需要可视化的技能，使用 Mermaid、Chart.js 等云端渲染工具
- 提供"高级版"：本地运行版本（需要用户自行部署），功能更强大

### 6.2 市场竞争风险

**风险 1**：扣子商店可能已经存在类似的 PM 技能，竞争激励

**应对措施**：
- **差异化定位**：强调"教学性"（ABC 原则），不仅生成交付物，还传授方法论
- **质量取胜**：参考原项目的"反模式"设计，提供更专业、更实用的技能
- **本土化优势**：针对中国 PM 的深度优化，是国外竞品无法比拟的
- **案例展示**：在扣子商店中展示真实案例（如"某互联网公司用此技能生成 PRD，节省 50% 时间"）

**风险 2**：用户可能更习惯使用 Claude Code 或本地工具，不愿意切换到扣子平台

**应对措施**：
- **多平台覆盖**：不仅上架扣子商店，还保持 GitHub、Claude Code 插件市场的更新
- **云端优势宣传**：强调扣子的"无需安装、开箱即用、随时随地使用"优势
- **免费试用**：提供免费 Skill，降低用户尝试门槛
- **用户教育**：通过内容营销（知乎文章、微信公众号）教育用户"为什么云端 AI 技能是未来趋势"

### 6.3 商业化风险

**风险 1**：付费技能下载量少，无法覆盖开发成本

**应对措施**：
- **免费+付费混合模式**：先提供免费技能，建立信任，再引导购买付费技能
- **打包定价**：提供"Professional Pack"等打包产品，提高客单价
- **限时促销**：定期推出限时折扣，刺激购买
- **退款保证**：提供"7 天无理由退款"，降低用户购买顾虑

**风险 2**：用户付费后，技能质量不达预期，导致差评和退款

**应对措施**：
- **严格质量控制**：每个技能上线前，必须经过至少 3 个真实用户测试
- **持续优化**：根据用户反馈，每周更新 1-2 个技能
- **透明度**：在扣子商店中明确说明技能的功能边界（如"本技能生成 PRD 初稿，仍需人工审核"）
- **客服支持**：提供企业微信或微信群客服，及时解决用户问题
