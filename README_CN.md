## 为什么要用这个技能库？

通用 AI 输出是产品经理最大的敌人。当你对 AI 说「写一份 PRD」却没有给出共同背景，你得到的只是一份没有任何利益相关者会信任、工程师无法执行的通用文档。

这个库给你和 AI 都提供相同的专业基础：每个方法论背后的**原理**、需要规避的**失败模式**，以及正确应用的**判断依据**。你不再重复解释，AI 不再瞎猜，工作质量切实提升。

**双重目标：功能性与教学性并重。** 技能让 AI 在专业水准上执行 PM 工作，同时也教会人类 PM 每个框架背后的「为什么」——让你能够解释它、调整它、传授给下一个人。两个目标缺一不可。

## 能帮你完成什么

按照你真正想达成的目标导航：

### 定义方向与战略
- [problem-framing-canvas](skills/problem-framing-canvas/SKILL.md) — MITRE 的「向内看 / 向外看 / 重新定框」序列；防止团队解决错误的问题
- [positioning-statement](skills/positioning-statement/SKILL.md) — Geoffrey Moore 的定位模板：定义你服务谁、解决什么问题、有何不同
- [product-strategy-session](skills/product-strategy-session/SKILL.md) — 完整战略弧线：定位 → 问题框架 → 解决方案探索 → 路线图（2-4 周）

### 利益相关方对齐
- [stakeholder-identification](skills/stakeholder-identification/SKILL.md) — 在接触任何人之前先绘制完整的利益相关方地图
- [stakeholder-mapping](skills/stakeholder-mapping/SKILL.md) — 运行两个互补矩阵（权力 × 利益；影响 × 权力）并对比找出差距
- [stakeholder-engagement-advisor](skills/stakeholder-engagement-advisor/SKILL.md) — 针对每个利益相关方的参与规划

### 用户调研与发现
- [discovery-interview-prep](skills/discovery-interview-prep/SKILL.md) — 基于妈妈测试法规划访谈
- [opportunity-solution-tree](skills/opportunity-solution-tree/SKILL.md) — 生成机会与解决方案，推荐优先验证的 POC
- [discovery-process](skills/discovery-process/SKILL.md) — 完整发现周期：框架 → 研究 → 综合 → 验证（3-4 周）
- **[chinese-user-research](skills/chinese-user-research/SKILL.md) 🇨🇳 新** — 中国用户研究方法论：微信生态调研、下沉市场用户访谈、用户分层模型

### 优先级与路线图
- [prioritization-advisor](skills/prioritization-advisor/SKILL.md) — 询问 3-5 个关于你上下文的问题，推荐 RICE、ICE、Kano 或合适替代方案
- [epic-breakdown-advisor](skills/epic-breakdown-advisor/SKILL.md) — 使用 Richard Lawrence 的 9 种模式拆分大 Epic
- [roadmap-planning](skills/roadmap-planning/SKILL.md) — 收集输入 → 定义 Epic → 优先排序 → 排期 → 传达（1-2 周）
- **[agile-sprint-planning](skills/agile-sprint-planning/SKILL.md) 🇨🇳 新** — 敏捷冲刺规划：适配中国互联网快节奏的 Sprint 管理方法

### 撰写 PM 交付物
- [user-story](skills/user-story/SKILL.md) — Mike Cohn 格式 + Gherkin 验收标准，含反模式
- [prd-development](skills/prd-development/SKILL.md) — 结构化 PRD：问题 → 用户画像 → 方案 → 度量 → 故事（2-4 天）
- [press-release](skills/press-release/SKILL.md) — 亚马逊逆向工作法：在写一行规格前先澄清产品愿景

### 验证与实验
- [pol-probe-advisor](skills/pol-probe-advisor/SKILL.md) — 根据假设和风险级别推荐原型类型
- [pol-probe](skills/pol-probe/SKILL.md) — 建设前记录轻量验证实验的模板
- **[ab-testing-advisor](skills/ab-testing-advisor/SKILL.md) 🇨🇳 新** — AB 测试决策顾问：从假设建立到统计显著性判断的完整决策流

### 增长与变现
- [business-health-diagnostic](skills/business-health-diagnostic/SKILL.md) — 使用真实指标诊断 SaaS 增长、留存、效率和资本健康度
- [organic-growth-advisor](skills/organic-growth-advisor/SKILL.md) — 麦肯锡增长金字塔分类
- [feature-investment-advisor](skills/feature-investment-advisor/SKILL.md) — 使用收入影响、成本、ROI 和战略价值做出构建/不构建决策

## 技能文件结构

每个 `SKILL.md` 遵循相同结构：

| 章节 | 包含内容 |
|---|---|
| **Frontmatter** | `name`、`description`、`type`、`intent`、`best_for`、`scenarios` |
| **Purpose（目标）** | 这个技能做什么，何时使用 |
| **Key Concepts（核心概念）** | 框架、定义、反模式——附词汇解释 |
| **Application（应用方法）** | AI 或人类可遵循的分步指导 |
| **Examples（示例）** | 展示推理过程的真实案例 |
| **Common Pitfalls（常见陷阱）** | 有名字的失败模式，含后果和纠正方法 |
| **References（参考资料）** | 相关技能和外部框架 |

## ABC 原则——始终保持教练姿态

这个库的核心精神：**AI Agent 使用一个技能完成任务，使用这个技能的人类 PM 也应该在完成时比开始时懂得更多。**

使用技能后你不只是得到一个产出物——你得到了产生这个产出物的推理框架，让你能够解释它、调整它、教给别人。
