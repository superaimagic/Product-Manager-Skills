## 快速开始

**第二步：引用技能文件**

在对话中直接告诉 AI：
```
参考 skills/okr-management/SKILL.md，帮我制定本季度产品团队的 OKR。
我们的产品是 [描述]，上季度主要问题是 [描述]。
```

**第三步：提供你的具体上下文**

技能框架提供方法论骨架，你的上下文让输出真正有价值：
- 产品类型和阶段
- 团队规模
- 当前面临的具体问题
- 已有的数据和约束

## 中文技能详解

### 1. OKR 管理 (`okr-management`)

**解决什么问题：**
- OKR 写成了 KPI 任务清单，不知道怎么写「关键结果」
- 季度结束 OKR 复盘流于形式
- KPI 考核文化强，OKR 推行阻力大

**触发场景：**
```
帮我基于 skills/okr-management/SKILL.md，
制定一套产品团队本季度的 OKR。
我们的公司目标是「提升用户留存」，产品是内容类 App，DAU 50 万。
```

---

### 2. 增长黑客 (`product-growth-hacking`)

**解决什么问题：**
- DAU 增长停滞，不知道卡在哪个漏斗环节
- 想做裂变活动但不知道怎么设计才有效
- 用户量大但变现难，增长和变现路径没打通

**触发场景：**
```
我的 App 月活 200 万，但 7 日留存只有 18%，
参考 skills/product-growth-hacking/SKILL.md 帮我诊断问题并给出策略。
```

---

### 3. AB 测试顾问 (`ab-testing-advisor`)

**解决什么问题：**
- 不知道样本量要多少才够，或者实验跑多久合适
- 数据出来了不知道发不发布
- 实验结果总是「不显著」，不知道原因

**触发场景：**
```
我想测试新手引导从 6 步缩短到 3 步的效果，
参考 skills/ab-testing-advisor/SKILL.md 帮我设计实验。
当前新用户 3 日激活率 35%，日均新用户 600 人。
```

---

### 4. 中国用户研究 (`chinese-user-research`)

**解决什么问题：**
- 微信群里收集反馈，用户都说「很好」，得不到真实反馈
- 不知道怎么招募下沉市场用户
- 访谈时用户顺着说，不能暴露真实痛点

**触发场景：**
```
我需要对我们 App 的三四线城市用户做 10 场深度访谈，
参考 skills/chinese-user-research/SKILL.md 帮我设计招募方案和访谈提纲。
```

---

### 5. 变现策略顾问 (`product-monetization-advisor`)

**解决什么问题：**
- 有流量但不知道怎么变现
- 做了会员但付费率很低，不知道哪里出了问题
- 广告收入下滑，需要找新变现路径

**触发场景：**
```
我们的工具类 App DAU 60 万，当前会员付费率 0.8%，
参考 skills/product-monetization-advisor/SKILL.md 诊断问题并给改进方案。
```

---

### 6. 敏捷冲刺规划 (`agile-sprint-planning`)

**解决什么问题：**
- Sprint 经常完不成，总是排太多
- Sprint 目标不清晰，只有任务列表
- 产品和研发在排期上总有摩擦

**触发场景：**
```
下周要开 Sprint 计划会，我有这些用户故事：[列表]，
参考 skills/agile-sprint-planning/SKILL.md 帮我规划 Sprint。
团队：4 后端 + 2 前端 + 1 测试，历史速度约 28 点。
```

## 与国际技能的配合使用

中文技能和国际技能可以组合使用，形成完整的 PM 工作流：

### 完整的产品立项流程

```
1. 用户研究       → chinese-user-research
2. 问题定义       → problem-statement + problem-framing-canvas  
3. 机会分析       → opportunity-solution-tree
4. 优先级决策     → prioritization-advisor
5. 目标设定       → okr-management
6. 需求文档       → prd-development
7. 故事拆解       → user-story + user-story-splitting
8. 迭代规划       → agile-sprint-planning
9. 上线后增长     → product-growth-hacking + ab-testing-advisor
10. 变现优化      → product-monetization-advisor
```

### 快速验证一个新功能想法

```
1. pol-probe-advisor → 选择验证方式
2. ab-testing-advisor → 设计实验
3. chinese-user-research → 定性补充
4. feature-investment-advisor → 决定是否全面开发
```

## 高级用法：场景提示词模板

将以下模板复制到你的 AI 对话中，替换 `[方括号]` 内容：

### OKR 制定
```
我是 [公司类型] 的产品经理，负责 [产品描述]。
参考 skills/okr-management/SKILL.md 的框架，
帮我为下个季度制定产品团队 OKR。

背景：
- 公司级目标：[描述]
- 上季度主要成果：[描述]
- 当前最大挑战：[描述]
- 团队规模：[描述]

请先帮我起草 1-2 个 Objective，然后为每个 O 设计 2-3 个 Key Result。
```

### 增长诊断
```
请参考 skills/product-growth-hacking/SKILL.md，
对我们的产品做 AARRR 增长诊断。

产品信息：
- 产品类型：[内容/工具/社交/电商等]
- DAU：[X]，MAU：[X]
- 新用户 7 日留存：[X%]
- 付费转化率：[X%]（如有）
- 主要流量来源：[描述]

请帮我找到最大的增长漏斗卡点，并给出 2-3 个具体的改进方向。
```

## 常见问题

**Q：技能文件是英文的，我可以用中文交互吗？**

A：可以。技能文件的框架是语言无关的，你用中文描述场景，AI 会用中文执行技能并给出中文输出。6 个中文技能文件本身就是中文写作。

**Q：怎么判断用哪个技能？**

A：按照上面的场景表找，或者直接告诉 AI「我想 [做什么]，这个项目里有合适的技能吗？」AI 会帮你匹配。

**Q：技能文件可以修改吗？**

A：可以，但建议先理解原有框架再修改。添加中国场景的例子是非常好的贡献方式。
