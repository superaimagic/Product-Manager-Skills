## 目的

确定你的SaaS商业模式是否从根本上可行且资本高效。用于计算单位经济效益、评估盈利能力、管理现金跑道以及决定何时扩展vs.优化。对融资、董事会汇报和做出明智投资权衡至关重要。

这不是财务报告工具——它是PM理解业务能否支撑增长、何时优先效率而非增长、以及哪些投资有正回报的框架。

## 核心概念

### 单位经济效益族

衡量客户级盈利能力的指标——可持续SaaS的基础。

**毛利率** — 扣除直接成本(COGS)后剩余的收入百分比。
- **为什么PM关注：** 一个在80%毛利率下产生¥100万收入的功能远比在30%毛利率下¥100万有价值。毛利率决定优先哪些功能。
- **公式：** `(收入 - COGS) / 收入 × 100`
- **COGS包括：** 服务器托管、基础设施、支付处理费、客户引导成本
- **基准：** SaaS 70-85%良好；<60%需关注

**CAC（客户获取成本）** — 获取一个客户的总成本。
- **为什么PM关注：** 塑造整个GTM战略。决定哪些渠道可行以及你能在产品驱动增长上投入多少。
- **公式：** `总销售与营销支出 / 新获取客户数`
- **基准：** 因模式差异——企业¥10万+可接受；中小企业<¥5,000为目标
- **包括：** 营销支出、销售工资、工具、佣金

**LTV（生命周期价值）** — 一个客户在其生命周期内预期产生的总收入。
- **为什么PM关注：** 告诉你获取方面能花多少。更高LTV使高端渠道和更长回本期可行。
- **公式（简单）：** `ARPU × 平均客户生命周期（月）`
- **公式（更好）：** `ARPU × 毛利率% / 流失率`
- **公式（高级）：** 考虑扩展、折现率、特定群组留存
- **基准：** 必须3倍+ CAC；因细分而异

**LTV:CAC比率** — 客户获取支出的效率。
- **为什么PM关注：** 增长可持续还是你在亏本买收入？决定何时扩展vs.优化。
- **公式：** `LTV / CAC`
- **基准：** 3:1健康；<1:1不可持续；>5:1可能投入不足
- **注意：** 此比率本身不讲述完整故事——还需要回本期

**回本期** — 从客户收入中回收CAC的月数。
- **为什么PM关注：** 现金效率。更快回本 = 更快再投入。慢回本即使在良好LTV:CAC下也可能扼杀增长。
- **公式：** `CAC / (月度ARPU × 毛利率%)`
- **基准：** <12个月优秀；12-18可接受；>24个月需关注
- **关键：** 必须有现金来支撑回本期

**贡献毛利** — 扣除所有可变成本后剩余的收入（不只是COGS）。
- **为什么PM关注：** 真实单位盈利能力。包括支持、处理费、可变运营支出。
- **公式：** `(收入 - 所有可变成本) / 收入 × 100`
- **可变成本：** COGS + 支持 + 支付处理 + 可变客户成功
- **基准：** SaaS 60-80%良好；<40%需关注

**毛利回本** — 使用实际利润而非收入的回本期。
- **为什么PM关注：** 比简单回本更准确。显示真实现金回收时间。
- **公式：** `CAC / (月度ARPU × 毛利率%)`
- **基准：** 通常比简单收入回本长1.5-2倍

**各渠道CAC回本** — 比较各获客渠道的回本。
- **为什么PM关注：** 渠道并非等价。基于回本效率优化渠道组合。
- **公式：** 分别计算各渠道的CAC和回本
- **用法：** 现金紧张时将预算分配给回本更快的渠道

### 资本效率族

衡量你使用现金增长的效率的指标。

**烧钱率** — 每月消耗的现金。
- **为什么PM关注：** 决定你能构建什么以及何时需要融资。高烧钱率要求激进收入增长。
- **公式（总烧钱）：** `月度现金支出（所有费用）`
- **公式（净烧钱）：** `月度现金支出 - 月度收入`
- **基准：** 早期阶段净烧钱<¥150万可管理；>¥350万需要清晰的收入路径

**跑道** — 现金耗尽的月数。
- **为什么PM关注：** 字面生存指标。决定里程碑、融资、盈利的时间线。
- **公式：** `现金余额 / 月度净烧钱`
- **基准：** 12+个月良好；6-12可管理；<6个月危机模式
- **规则：** 在6-9个月跑道时融资，不是3个月

**运营支出(OpEx)** — 运营业务的成本（不含COGS）。
- **为什么PM关注：** 你团队的工资在这里。 downturn时"效率"削减发生的地方。
- **类别：** 销售与营销(S&M)、研发(R&D)、管理与行政(G&A)
- **基准：** 随规模增长应比收入慢（运营杠杆）

**净利润（利润率）** — 所有费用后的实际盈亏。
- **为什么PM关注：** 真实底线。你在赚钱吗？你能自融资增长吗？
- **公式：** `收入 - 所有费用（COGS + OpEx）`
- **基准：** 早期SaaS常为负（增长模式）；成熟应为10-20%+利润率

**营运资本影响** — 收入确认与现金收取之间的现金时间差。
- **为什么PM关注：** 年度合同预付提振现金。月付延迟现金。影响跑道计算。
- **示例：** ¥100万年度合同预付 = 现在¥100万现金，而非每月¥8.3万
- **用法：** 规划跑道时理解现金vs.收入时间差

---

### 效率比率族

衡量增长vs.盈利权衡的复合指标。

**40法则** — 增长率 + 利润率应超过40%。
- **为什么PM关注：** 平衡增长vs.效率的框架。指导何时优先盈利而非增长。
- **公式：** `收入增长率% + 利润率%`
- **基准：** >40健康；25-40可接受；<25需关注
- **示例：** 60%增长 + (-20%)利润率 = 40（健康的增长模式SaaS）
- **示例：** 20%增长 + 25%利润率 = 45（健康的成熟SaaS）

**魔力数字** — 销售与营销效率。
- **为什么PM关注：** 你的GTM引擎在工作吗？应该加大投入还是先优化？
- **公式：** `(本季度收入 - 上季度收入) × 4 / 上季度S&M支出`
- **基准：** >0.75高效；0.5-0.75可接受；<0.5先修复再扩展
- **注意：** "× 4"将季度收入变化年化

**运营杠杆** — 收入增长vs.成本增长的比较。
- **为什么PM关注：** 你在高效扩展吗？收入应该比成本增长更快。
- **衡量：** 收入增长率vs. OpEx增长率随时间
- **良好：** 收入增长50%，OpEx增长30%（正杠杆）
- **不良：** 收入增长20%，OpEx增长40%（负杠杆）

**单位经济效益** — 每个"单位"（客户、席位、交易）盈利能力的通用术语。
- **为什么PM关注：** 商业模式在单位层面从根本上可行吗？
- **计算：** 每单位收入 - 每单位成本
- **要求：** 需要正贡献；目标所有可变成本后>$0

---

### 反模式（这不是什么）

- **不是虚荣指标：** 高LTV如果回本期4年而客户3年流失就毫无意义。
- **不是静态基准：** "好的"CAC因商业模式差异巨大（PLG vs.企业销售）。
- **不是孤立数字：** LTV:CAC比率没有回本期可能误导（好的比率，差的现金效率）。
- **不只是财务的问题：** PM必须负责单位经济效益——每个功能决策影响毛利率和CAC。

---

### 何时使用这些指标

**在以下情况使用：**
- 评估是否扩大获客（LTV:CAC、回本、魔力数字）
- 决定功能投资（毛利影响、对LTV的贡献）
- 规划跑道和融资（烧钱率、跑道、40法则）
- 比较客户细分或渠道（按细分的单位经济效益）
- 董事会/投资者汇报（40法则、魔力数字、LTV:CAC）
- 在增长和盈利之间选择（40法则权衡）

**在以下情况不要使用：**
- 没有收入上下文做决策（配合 `saas-revenue-growth-metrics`)
- 没有标准化直接比较截然不同的商业模式
- 早期产品发现（收入前聚焦PMF而非单位经济效益）
- 短期战术决策（用参与度指标而非LTV）

## 应用

### 步骤1：计算单位经济效益

使用 `template.md` 中的模板计算你的单位经济效益指标。

#### 毛利率
```
毛利率 = (收入 - COGS) / 收入 × 100

COGS包括：
- 服务器托管与基础设施成本
- 支付处理费
- 客户引导成本
- 直接交付成本
```

**示例：**
- 收入：¥1,000,000
- COGS：¥200,000（托管¥120万，处理¥50万，引导¥30万）
- 毛利率 = (¥100万 - ¥20万) / ¥100万 = 80%

**质量检查：**
- 毛利率随规模改善吗？（应受益于规模经济）
- 哪些产品/功能毛利率最高？（优先那些）
- 毛利率>70%吗？（SaaS应该是高毛利）

---

#### CAC（客户获取成本）
```
CAC = 总销售与营销支出 / 新获取客户数

S&M支出包括：
- 营销工资与工具
- 销售工资与佣金
- 广告与付费渠道
- SDR/BDR团队成本
```

**示例：**
- 销售与营销支出：¥500,000/月
- 新客户：100/月
- CAC = ¥500,000 / 100 = ¥5,000

**质量检查：**
- CAC各渠道一致吗？（按渠道计算）
- CAC随时间增加还是减少？（应随规模下降）
- CAC因客户细分差异吗？（中小企业 vs. 企业）

---

#### LTV（生命周期价值）
```
LTV（简单） = ARPU × 平均客户生命周期（月）

LTV（更好） = ARPU × 毛利率% / 月流失率

LTV（高级） = 考虑扩展、群组特定留存、折现率
```

**示例（简单）：**
- ARPU：¥500/月
- 平均生命周期：36个月
- LTV = ¥500 × 36 = ¥18,000

**示例（更好）：**
- ARPU：¥500/月
- 毛利率：80%
- 月流失：2%
- LTV = (¥500 × 80%) / 2% = ¥400 / 0.02 = ¥20,000

**质量检查：**
- LTV随时间增长吗？（来自扩展、改善留存）
- LTV因群组差异吗？（新客户更/更有价值？）
- LTV因细分差异吗？（企业 vs. 中企业）

---

#### LTV:CAC比率
```
LTV:CAC比率 = LTV / CAC
```

**示例：**
- LTV：¥20,000
- CAC：¥5,000
- LTV:CAC = ¥20,000 / ¥5,000 = 4:1

**质量检查：**
- 比率>3:1吗？（可持续增长最低要求）
- 比率>5:1吗？（可能投入不足）
- 比率随时间改善还是恶化？

**解读：**
- **<1:1** = 每个客户都在亏钱（不可持续）
- **1-3:1** = 边际经济效益（优化后再扩展）
- **3-5:1** = 健康（放心扩展）
- **>5:1** = 可能投入不足（可以增长更快）

---

#### 回本期
```
回本期（月） = CAC / (月度ARPU × 毛利率%)
```

**示例：**
- CAC：¥5,000
- 月度ARPU：¥500
- 毛利率：80%
- 回本 = ¥5,000 / (¥500 × 80%) = ¥5,000 / ¥400 = 12.5个月

**质量检查：**
- 回本<12个月吗？（优秀）
- 回本<18个月吗？（可接受）
- 你有现金跑道来支撑回本期吗？

**关键洞察：** 4:1 LTV:CAC搭配36个月回本是现金陷阱。3:1 LTV:CAC搭配8个月回本对增长更好。

---

#### 贡献毛利
```
贡献毛利 = (收入 - 所有可变成本) / 收入 × 100

可变成本包括：
- COGS
- 支持成本（可变部分）
- 支付处理
- 可变客户成功成本
```

**示例：**
- 收入：¥1,000,000
- COGS：¥200,000
- 可变支持：¥50,000
- 支付处理：¥30,000
- 贡献毛利 = (¥100万 - ¥28万) / ¥100万 = 72%

**质量检查：**
- 贡献毛利>60%吗？（SaaS良好）
- 某些产品/细分毛利率更低吗？（考虑淘汰）
- 毛利率随规模改善吗？

---

### 步骤2：计算资本效率

#### 烧钱率
```
总烧钱率 = 月度总现金支出
净烧钱率 = 月度总现金支出 - 月度收入
```

**示例：**
- 月度费用：¥800,000
- 月度收入：¥400,000
- 总烧钱：¥800,000/月
- 净烧钱：¥400,000/月

**质量检查：**
- 净烧钱随时间减少吗？（通往盈利的路径）
- 烧钱率在跑道范围内可持续吗？
- 烧钱率相对于收入是多少？（烧钱倍数）

---

#### 跑道
```
跑道（月） = 现金余额 / 月度净烧钱
```

**示例：**
- 现金余额：¥6,000,000
- 净烧钱：¥400,000/月
- 跑道 = ¥600万 / ¥40万 = 15个月

**质量检查：**
- 跑道>12个月吗？（健康）
- 跑道<6个月吗？（危机——现在融资或削减烧钱）
- 你能在跑道结束前达到下一个里程碑吗？

**规则：** 在6-9个月跑道时开始融资，不是3个月。

---

#### 运营支出(OpEx)
```
OpEx = 销售与营销 + 研发 + 管理与行政

追踪收入占比：
S&M收入占比
R&D收入占比
G&A收入占比
```

**示例：**
- 收入：¥1000万/年
- S&M：¥500万（收入50%）
- R&D：¥300万（收入30%）
- G&A：¥100万（收入10%）
- 总OpEx：¥900万（收入90%）

**质量检查：**
- OpEx类别增长比收入慢吗？（运营杠杆）
- S&M支出高效吗？（检查魔力数字）
- G&A<收入15%吗？（应保持低）

---

#### 净利润（利润率）
```
净利润 = 收入 - COGS - OpEx
利润率% = 净利润 / 收入 × 100
```

**示例：**
- 收入：¥1000万
- COGS：¥200万
- OpEx：¥900万
- 净利润 = ¥1000万 - ¥200万 - ¥900万 = -¥100万（亏损）
- 利润率 = -10%

**质量检查：**
- 利润率随时间改善吗？（通往盈利的路径）
- 以当前增长率何时盈亏平衡？
- 你的亏损是在增长方面投资吗？（如果LTV:CAC健康可接受）

---

### 步骤3：计算效率比率

#### 40法则
```
40法则 = 收入增长率% + 利润率%
```

**示例1（增长模式）：**
- 收入增长：80% YoY
- 利润率：-30%
- 40法则 = 80% + (-30%) = 50 ✅ 健康

**示例2（成熟）：**
- 收入增长：25% YoY
- 利润率：20%
- 40法则 = 25% + 20% = 45 ✅ 健康

**示例3（问题）：**
- 收入增长：30% YoY
- 利润率：-35%
- 40法则 = 30% + (-35%) = -5 🚨 不健康

**质量检查：**
- 40法则>40吗？（健康平衡）
- 40法则>25吗？（可接受）
- 40法则<25吗？（烧钱但没有足够增长）

**权衡：**
- 早期：最大化增长，接受亏损（60%增长，-20%利润率 = 40）
- 增长期：平衡（40%增长，5%利润率 = 45）
- 成熟期：优先盈利（20%增长，25%利润率 = 45）

---

#### 魔力数字
```
魔力数字 = (本季度收入 - 上季度收入) × 4 / 上季度S&M支出
```

**示例：**
- Q2收入：¥250万
- Q1收入：¥200万
- Q1 S&M支出：¥80万
- 魔力数字 = (¥250万 - ¥200万) × 4 / ¥80万 = ¥200万 / ¥80万 = 2.5

**质量检查：**
- 魔力数字>0.75吗？（高效——加大S&M投入）
- 魔力数字0.5-0.75吗？（可接受——优化后再扩展）
- 魔力数字<0.5吗？（低效——先修复GTM）

**解读：**
- **>1.0** = 每¥1 S&M投入获得¥1+新ARR（优秀）
- **0.75-1.0** = 高效，放心扩展
- **0.5-0.75** = 边际，先优化再扩展
- **<0.5** = 低效，先修复再投入更多

---

#### 运营杠杆
随时间追踪来看你是否高效扩展。

**示例：**
| 季度 | 收入 | YoY增长 | OpEx | YoY增长 | 杠杆 |
|------|------|---------|------|---------|------|
| Q1 2024 | ¥800万 | - | ¥600万 | - | - |
| Q2 2024 | ¥1000万 | 25% | ¥700万 | 17% | 正 ✅ |
| Q3 2024 | ¥1200万 | 20% | ¥900万 | 29% | 负 ⚠️ |

**质量检查：**
- 收入增长比OpEx快吗？（正杠杆）
- 你相对于收入扩展OpEx太快了吗？
- 哪个OpEx类别增长最快？（R&D、S&M、G&A）

---

### 步骤4：按细分和渠道分析

**单位经济效益因细分差异巨大：**

| 细分 | CAC | LTV | LTV:CAC | 回本期 | 毛利率 |
|------|-----|-----|---------|--------|--------|
| 中小企业 | ¥3,500 | ¥14,000 | 4:1 | 8个月 | 75% |
| 中市场 | ¥35,000 | ¥175,000 | 5:1 | 12个月 | 80% |
| 企业 | ¥350,000 | ¥210万 | 6:1 | 24个月 | 85% |

**质量检查：**
- 哪个细分单位经济效益最好？
- 哪个细分回本最快？（现金紧张时优先）
- 哪个细分LTV最高？（投资留存/扩展）

### 示例2：好的LTV:CAC，差的回本（现金陷阱）

**公司：** 企业CRM（企业销售模式）

**单位经济效益：**
- CAC：¥56万
- LTV：¥280万
- LTV:CAC：5:1 ✅（看起来很好！）
- 回本期：36个月 🚨（糟糕！）
- 毛利率：85%

**资本效率：**
- 月度净烧钱：¥1400万
- 跑道：9个月 🚨
- 平均客户生命周期：48个月
- 平均合同：¥70万/年

**分析：**
- ⚠️ 好的LTV:CAC比率（5:1）掩盖了现金问题
- 🚨 36个月回本搭配9个月跑道 = 现金陷阱
- 🚨 需要3年回收CAC，但只有9个月现金
- ⚠️ 客户留存4年，所以如果有现金经济效益可行

**问题：** 你会在回收获取成本之前耗尽现金。

**行动：**
1. 协商预付年度付款（将回本降至12个月）
2. 融资以延长跑道（需要36+月来支撑增长）
3. 降低CAC（缩短销售周期、改善转化）
4. 瞄准更快回本的更小交易（中市场 vs. 企业）

### 示例3：扩展过快（负运营杠杆）

**季度趋势：**
| 季度 | 收入 | OpEx | 净利润 | 收入增长 | OpEx增长 |
|------|------|------|--------|---------|---------|
| Q1 | ¥700万 | ¥560万 | -¥560万 | - | - |
| Q2 | ¥910万 | ¥840万 | -¥840万 | 30% | 50% 🚨 |
| Q3 | ¥1120万 | ¥1260万 | -¥1260万 | 23% | 50% 🚨 |

**分析：**
- 🚨 OpEx增长比收入更快（50% vs. 23-30%）
- 🚨 亏损加速（¥560万 → ¥1260万 两个季度内）
- 🚨 负运营杠杆（应为正）
- ⚠️ 扩展S&M和R&D但没有相应收入增长

**问题：** 烧钱更快而收入增长放缓。

**行动：**
1. 冻结招聘直到收入跟上
2. 削减低效S&M支出（魔力数字可能<0.5）
3. 先改善单位经济效益再扩展
4. 目标OpEx增长<收入增长

## 常见陷阱

### 陷阱1：庆祝高LTV但不检查回本
**症状：** "我们的LTV:CAC是6:1，太棒了！"

**后果：** 6:1比率搭配48个月回本是现金陷阱。你会在回收CAC之前耗尽现金。

**修复：** 总是将LTV:CAC搭配回本期。3:1搭配10个月回本胜过6:1搭配36个月回本。

---

### 陷阱2：计算LTV时忽略毛利率
**症状：** "LTV = ¥700/月 × 36个月 = ¥25,200"

**后果：** 你用的是收入而非利润。30% COGS后的实际LTV = ¥17,640而非¥25,200。

**修复：** 在LTV计算中始终包含毛利率。`LTV = ARPU × 毛利率% / 流失率`。

---

### 陷阱3：低魔力数字下扩展S&M
**症状：** "我们需要更快增长——加倍S&M投入！"（魔力数字 = 0.3）

**后果：** 你在给坏引擎加油。加倍投入只会加速烧钱而没有相应的收入增长。

**修复：** 只有当魔力数字>0.75时才扩展S&M。如果<0.5，先修复GTM效率。

---

### 陷阱4：使用简单LTV公式
**症状：** "LTV = ARPU × 生命周期"（忽略扩展、折现率、群组差异）

**后果：** 决策中高估LTV。现实：扩展提升LTV；折扣降低它；群组有差异。

**修复：** 大决策用复杂LTV模型。简单LTV仅用于方向性指引。

---

### 陷阱5：忘记货币时间价值
**症状：** "¥10万今天收入 = ¥10万5年后收入"

**后果：** 对长回本业务高估LTV。5年后的¥10万在5%折现率下≈¥7.8万今天的价值。

**修复：** 对LTV期间>24个月的折现未来现金流。使用NPV（净现值）。

---

### 陷阱6：跨不同回本期比较CAC
**症状：** "渠道A CAC ¥3.5万，渠道B CAC ¥5.6万——渠道A更好！"

**后果：** 如果渠道A有24个月回本而渠道B有8个月回本，渠道B实际上更好（更快现金回收）。

**修复：** 搭配CAC + 回本比较，而非孤立看CAC。

---

### 陷阱7：庆祝40法则>40但负现金流
**症状：** "40法则 = 50，我们表现太好了！"（60%增长，-10%利润率，月烧¥3500万）

**后果：** 40法则不考虑绝对烧钱。你可能平衡很好但只有3个月跑道。

**修复：** 搭配40法则和烧钱率及跑道。平衡重要，但生存更重要。

---

### 陷阱8：忽略细分特定的单位经济效益
**症状：** "混合CAC ¥1.4万，混合LTV ¥7万，我们很好！"

**后果：** 中企业细分可能有¥3,500 CAC / ¥1.4万 LTV（好），而企业有¥14万 CAC / ¥10.5万 LTV（差）。混合指标隐藏问题。

**修复：** 按细分计算单位经济效益。独立优化每个细分。

---

### 陷阱9：混淆毛利率与贡献毛利
**症状：** "毛利率80%，我们的利润率很棒！"

**后果：** 扣除可变支持成本(10%)和支付处理(3%)后，贡献毛利可能是67%——而非80%。

**修复：** 追踪毛利率（仅COGS）和贡献毛利（所有可变成本）。用贡献毛利算单位经济效益。

---

### 陷阱10：忘记营运资本时间差
**症状：** "基于烧钱率我们有12个月跑道"（但所有合同都是月付）

**后果：** 预付年度合同暂时提振现金。月付延迟现金收取。跑道比烧钱率暗示的长/短。

**修复：** 计算跑道时考虑营运资本。现金跑道 ≠ 收入跑道。
