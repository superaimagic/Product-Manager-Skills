## 目标（Purpose）

中国用户研究不是把欧美方法直接翻译一遍——中国用户的沟通方式、信任建立机制、对产品反馈的表达习惯都有其独特性。

最大的陷阱：**中国用户倾向于给访谈者「想要听到的答案」**。这不是他们在撒谎，而是文化中的礼貌规范——直接批评陌生人的产品会让对方不舒服，所以他们会说「挺好的」。

破解这个问题是中国用户研究的核心技能。

---

## 核心概念（Key Concepts）

### 中国用户的五个行为特征

**1. 微信中心化**
- 微信是中国用户的主要通信工具，也是信息获取、娱乐、支付的核心入口
- 研究含义：招募、调研、反馈收集最好在微信生态内进行，门槛最低
- 具体工具：微信问卷（腾讯问卷）、微信小程序内调研、社群讨论

**2. 「面子」与社交压力**
- 在群体场合（焦点小组）批评一个产品可能让用户感觉在批评介绍者
- 研究含义：1:1 深度访谈比焦点小组更能获得真实反馈；匿名问卷比实名问卷更诚实

**3. 圈层分化**
中国用户不是同质的，圈层差异巨大：

| 圈层 | 特征 | 研究方式 |
|------|------|---------|
| 一线城市白领 | 时间敏感、效率导向、国际化视野 | 付费访谈（1-2 小时），问题可以较深 |
| 二线城市中产 | 价格敏感、品质意识上升、国内平台重度用户 | 混合：问卷 + 访谈 |
| 下沉市场（三线以下） | 时间充裕、口碑传播、微信视频号/快手重度用户 | 社群聊天式调研（更自然） |
| 银发群体（50+） | 语音使用为主、对复杂操作有挫败感、家庭场景驱动 | 面对面，用通俗语言 |
| Z世代（95后00后） | 短视频重度用户、二次元文化、抗拒「说教式」产品 | 社媒分析 + 社群观察 |

**4. 实用主义先行**
- 中国用户对「有用不有用」的判断比「好不好看」更优先
- 研究含义：问「这帮你解决了什么问题」比问「你觉得设计怎么样」更有效

**5. 价格敏感度分层**
- 免费 vs 付费的心理账户差异显著
- 研究含义：测试付费意愿时要区分「会不会用」和「会不会付钱」，这是两个完全不同的问题

### 妈妈测试（Mom Test）的中国版适配

妈妈测试原则（Rob Fitzpatrick）：问过去行为，不问对未来的意见。

**原版问题（无效）：** 「如果我们做了 xxx 功能，你会用吗？」

**中国用户的额外挑战：** 即使你问过去行为，用户也可能因为礼貌夸大正面描述。

**中国版强化版妈妈测试原则：**
1. **问具体事件，不问一般习惯**
   - 差：「你会经常分享内容给朋友吗？」
   - 好：「上周你有没有主动把什么东西分享给朋友过？当时是什么情况？」

2. **问最近的一次行为**
   - 「最近一次你 [完成了某个任务]，是什么时候？你怎么做的？」

3. **问痛苦，不问评分**
   - 差：「你觉得我们的产品哪里可以改进？」
   - 好：「上次你用的时候，有没有什么地方卡住了，当时是什么感受？」

4. **用沉默制造空间**
   - 问完问题后，不要急着补充解释；沉默 5-10 秒，用户会自然填补空白，通常是最真实的想法

### 反模式（Anti-Patterns）

❌ **领导式提问**
- 差：「我们觉得这个功能很好用，你觉得呢？」
- 好：「这个功能你上次用是什么场景？」

❌ **在微信群里问「觉得 xxx 怎么样」**
- 群里会有首因效应（第一个回答影响其他人）和从众压力
- 解法：匿名收集或 1:1 私信

❌ **把「用户要求的功能」直接加进产品**
- 中国用户说「我想要 xxx 功能」，真正的问题往往是「我在完成 yyy 目标时遇到了障碍」
- 解法：问「为什么你需要这个功能，是要完成什么任务？」

## 应用方法（Application）

### 模块 1：用户招募

**方式一：现有用户招募（最推荐）**

渠道：
- App 内弹窗：「帮我们改进产品，参与 20 分钟访谈，获得 [奖励]」
- 客服系统触达：联系近期有反馈的用户
- 后台筛选：根据使用行为标签（高频用户、已流失用户、付费用户）定向邀请

邀请消息模板（微信私信）：
```
你好！我是 [产品名] 的产品团队成员 [你的名字]。
看到你最近经常使用 [某功能]，想请你聊 20 分钟，
聊聊你的真实使用感受，帮我们把产品做得更好。
[奖励说明：微信红包 XX 元/京东卡/平台会员]
感兴趣的话，回复「感兴趣」，我们约个时间？
```

**方式二：微信群招募**

在自己运营的用户群或合作 KOC 的群里招募：
- 说明访谈目的（改进产品，不是卖东西）
- 提供明确奖励（现金红包比积分更有效）
- 私信有意向的用户，不要在群里做访谈

**方式三：专业调研平台**

- 见数（腾讯旗下，适合大规模定量调研）
- 一起调研（专业调研社区）
- 猫爪调研（垂直领域用户社区）

注意：平台招募的用户存在「职业问卷用户」风险，需要加筛选题

**用户筛选问卷（3-5 题）：**
```markdown
1. 你在过去 30 天内用过 [产品名] 吗？
   □ 是，每天用  □ 是，每周 2-3 次  □ 是，偶尔用  □ 没有用过

2. 你最近一次用 [产品名]，主要是做什么？（开放填写）

3. 你在使用 [产品名] 时有没有遇到过让你不爽的地方？
   □ 有，很多  □ 有，偶尔  □ 没有

（选 1、2 和 3 答「有」的用户优先入选）
```

### 模块 2：访谈设计

**访谈结构（45 分钟）**

**开场（5 分钟）**
```
「你好！非常感谢你今天来参加访谈。
我们今天聊的不是测试，也没有标准答案——我们只是想了解你真实的使用体验。
你说什么都不会影响到我们的判断，越真实越有价值。
我可能会问一些看起来很基础的问题，那是因为我们真的想从你的角度理解这件事。
整个过程大概 45 分钟。你同意我们录音用于内部研究分析吗？（不会对外公开）」
```

**挖掘背景（10 分钟）**
```
1. 「你每天主要用哪些 App？[产品名] 在你手机里算什么地位？」
2. 「上周你用 [产品名] 最主要是用来做什么？」
3. 「除了 [产品名]，你还会用什么来完成这件事？」
```

**深挖具体事件（20 分钟）**
```
4. 「跟我说一下，最近一次你用 [产品名] 完成了 [某任务]，是什么情况？」
   → 继续追问：「然后呢？」「那时候你是什么感受？」
   → 继续追问：「有什么地方让你觉得麻烦或者卡住了吗？」

5. 「有没有你本来想做，但发现做不到、或者很难做到的事情？」
   → 追问：「能描述一下当时的情况吗？」

6. 「如果你可以改 [产品名] 的一件事，你会改什么？为什么？」
   → 这是可以直接问的，但要追问「为什么」
```

**探索替代品（5 分钟）**
```
7. 「如果 [产品名] 突然消失了，你会怎么解决这个问题？」
   → 这个问题能揭示产品的真实不可替代性
```

**结尾（5 分钟）**
```
8. 「还有什么你觉得我们应该知道但我没问到的？」
（开放空间，经常有意外收获）
```

### 模块 3：下沉市场专项调研

下沉市场（三四五线城市、县城）用户的访谈特别注意：

**语言适配：**
- 避免「用户体验」「产品逻辑」「需求」等互联网黑话
- 用「用起来顺不顺手」代替「用户体验如何」
- 用「你一般怎么解决这个问题」代替「你的需求是什么」

**访谈场景：**
- 下沉用户更信任「熟人介绍」——尽量通过当地 KOC 或微信群领袖来建立信任
- 倾向于在熟悉的环境（家里、社区广场）接受访谈，而非正式的会议室

**核心问题调整（下沉市场版）：**
```
「你们那边大家现在都在用什么 App，为什么？」
「你第一次知道 [产品名] 是怎么知道的？」
「你用 [产品名] 最主要是打发时间，还是有什么具体的事要做？」
「家里人或者邻居有没有也在用的？你们有没有互相推荐过？」
```

---

### 模块 4：问卷设计（定量）

**适合问卷的场景：**
- 验证定性访谈发现的规律（「有多少人有这个问题？」）
- 覆盖大量用户获取代表性数据
- 不适合：挖掘深层动机（用访谈）

**中国用户问卷的 5 条设计原则：**

1. **短**：最多 10 题，3 分钟能填完；否则完成率暴跌
2. **具体**：「您多久使用一次这个功能？」比「您满意度如何？」更有价值
3. **包含负向选项**：问「哪里让你不爽」比只问「哪里让你满意」更能获得真实反馈
4. **结合选择题和开放填写**：纯选择题会导致思维被限制在你提供的选项里
5. **奖励在先**：告诉用户奖励，而不是填完才说

**NPS 问卷在中国的注意事项：**
- 中国用户给 9-10 分的比例天然高于欧美（礼貌文化）
- NPS 绝对数值不重要，重要的是变化趋势和开放题的内容
- 必须配合「你最主要给这个分数的原因是？」的开放题

## 示例（Examples）

### 示例 1：某社区 App 的下沉市场研究

**背景：** App 在三四线城市意外爆火，团队想理解这群用户的真实使用场景

**招募方式：** 通过已有用户发现活跃的「社区团购群主」，请他们帮忙推荐 3-5 位活跃用户

**关键访谈发现（通过「最近一次」法）：**

用户 A（四线城市全职妈妈）：
> 「上周我在上面认识了对面楼的一个妈妈，孩子同岁的，现在经常带孩子一起玩。你们的 App 就是个找邻居的地方嘛。」

用户 B（三线城市退休大爷）：
> 「我就是每天上去看看本地有什么新鲜事，比看新闻联播有意思，那些都是本地的人说本地的事。」

**洞察：** 这群用户把产品当「线上邻里社区」用，核心价值是「本地感」和「社交连接」，而非团队原来以为的「信息获取」。

**后续动作：** 将「同楼/同小区」的社交关系链作为核心功能深化，而非做成通用内容平台。

---

### 示例 2：解码「用户说好用但不再用了」

**背景：** 问卷显示用户满意度 4.2/5，但 30 日留存只有 22%

**问题诊断：** 问卷数据不可信——用户在问卷里说「满意」，实际上已经流失了

**研究方法调整：**
- 改用「最近一次打开是什么时候，当时做了什么」的访谈法
- 发现：用户的「满意」是「功能上没问题」，但不开 App 是因为「没有打开的理由」——缺乏推送机制和社交召回

**关键洞察：** 满意度高不代表留存好，留存靠的是「理由」不是「评分」。

## 常见陷阱（Common Pitfalls）

### 陷阱 1：把微信群讨论当定性研究

**症状：** 在用户群里问「大家觉得新功能怎么样」，得到 90% 好评，就认为验证成功了

**后果：** 群里的回复受首因效应和社交压力影响，不代表真实意见

**解法：** 群讨论用于「收集建议线索」，不用于「验证假设」；验证要用 1:1 访谈或匿名问卷

---

### 陷阱 2：只访谈「好用户」

**症状：** 总是邀请活跃的高净值用户参与访谈，获得的都是正面反馈

**后果：** 错过了导致用户流失的真实体验；幸存者偏差

**解法：** 刻意招募「流失用户」（30 天以上未活跃）和「低频用户」，他们的反馈最有价值

---

### 陷阱 3：把「会用」和「会推荐」当做相同指标

**症状：** 用 NPS 分数高来证明产品好

**后果：** 中国用户的 NPS 评分受礼貌文化影响显著虚高，无法作为独立指标

**解法：** NPS 必须配合「为什么给这个分数」的追问；更可信的指标是实际分享行为数据
