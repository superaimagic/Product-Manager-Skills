## 目的

任何SaaS财务指标的快速参考，无需深度教学。适用于需要快速公式查询、基准检查或决策框架提醒的场景。关于详细解释、计算和示例，请参见相关的深度技能。

这不是教学工具——而是一份为速度优化的速查表。浏览、找到、应用。

## 核心概念

### 指标分类

指标分为四大类：

1. **收入与增长** — 顶线收入（收入、ARPU、ARPA、MRR/ARR、流失、NRR、扩展）
2. **单位经济** — 客户级别盈利性（CAC、LTV、回收期、利润率）
3. **资本效率** — 现金管理（烧钱率、跑道、运营支出、净利润）
4. **效率比率** — 增长与盈利平衡（40法则、Magic Number）

### 何时使用本技能

**使用时机：**
- 你需要快速的公式或基准
- 你正在准备董事会或投资者电话会议
- 你正在评估一个决策，需要检查哪些指标重要
- 你想快速识别红旗指标

**不要使用时机：**
- 你需要详细的计算指导（使用 `saas-revenue-growth-metrics` 或 `saas-economics-efficiency-metrics`）
- 你第一次学习这些指标（从深度技能开始）
- 你需要示例和常见陷阱（相关技能中有涵盖）

## 应用

### 所有指标参考表

| **指标** | **公式** | **它衡量什么** | **好的基准** | **红旗** |
|------------|-------------|----------------------|-------------------|--------------|
| **收入** | 费用前的总销售额 | 赚取的顶线收入 | 年同比增长率 >20%（因阶段而异） | 收入增速慢于成本增速 |
| **ARPU** | 总收入 / 总用户数 | 每个用户的收入 | 因模式而异；追踪趋势 | ARPU逐队列下降 |
| **ARPA** | MRR / 活跃账户数 | 每个客户账户的收入 | SMB：¥100-¥1K；中型：¥1K-¥10K；企业：¥10K+ | 高ARPA + 低ARPU（席位数未充分变现） |
| **ACV** | 每合同年度经常性收入 | 年化合同价值 | SMB：¥5K-¥25K；中型：¥25K-¥100K；企业：¥100K+ | ACV下降（无意中向下游移动） |
| **MRR/ARR** | MRR × 12 = ARR | 可预测的经常性收入 | 增长 + 质量重要；追踪各组成部分 | 新MRR下降而流失稳定/增长 |
| **流失率** | 流失客户数 / 期初客户数 | 取消的客户百分比 | 月度 <2% 优秀，<5% 尚可；年度 <10% 优秀 | 流失率逐队列增加 |
| **NRR** | (期初ARR + 扩展 - 流失 - 缩减) / 期初ARR × 100 | 收入留存 + 扩展 | >120% 卓越；100-120% 良好；90-100% 尚可 | NRR <100%（基础在缩减） |
| **扩展收入** | 向上销售 + 交叉销售 + 用量增长 | 来自现有客户的额外收入 | 总收入的20-30% | 扩展收入 <MRR的10% |
| **Quick Ratio** | (新MRR + 扩展MRR) / (流失MRR + 缩减MRR) | 收入增长 vs. 损失 | >4 卓越；2-4 健康；<2 漏水桶 | Quick Ratio <2（漏水桶） |
| **毛利率** | (收入 - COGS) / 收入 × 100 | 扣除直接成本后的收入百分比 | SaaS：70-85% 良好；<60% 堪忧 | 毛利率 <60% 或持续下降 |
| **CAC** | 总销售和营销支出 / 新客户数 | 获取一个客户的成本 | 因阶段而异：企业 ¥10K+ 可接受；SMB <¥500 | CAC上升而LTV持平 |
| **LTV** | ARPU × 毛利率% / 流失率 | 来自一个客户的总收入 | 必须是CAC的3倍+；因细分市场而异 | LTV逐队列下降 |
| **LTV:CAC** | LTV / CAC | 单位经济效率 | 3:1 健康；<1:1 不可持续；>5:1 投资不足 | LTV:CAC <1.5:1 |
| **回收期** | CAC / (月度ARPU × 毛利率%) | 回收CAC的月数 | <12个月 优秀；12-18 可接受；>24 堪忧 | 回收期 >24个月（现金陷阱） |
| **边际贡献率** | (收入 - 所有可变成本) / 收入 × 100 | 扣除可变成本后的真实贡献 | SaaS：60-80% 良好；<40% 堪忧 | 边际贡献率 <40% |
| **烧钱率** | 月度现金支出 - 收入 | 每月消耗的现金 | 早期净烧钱 <¥200K 可控；<¥500K 可增长 | 净烧钱在加速 |
| **跑道** | 现金余额 / 月度净烧钱 | 资金用完前的月数 | 12+个月 良好；6-12 可接受；<6 危机 | 跑道 <6个月 |
| **运营支出** | 销售营销 + 研发 + 行政 | 运营业务的成本 | 应比收入增长慢 | 运营支出增速快于收入 |
| **净利润** | 收入 - 所有费用 | 实际利润/亏损 | 早期负数可接受；成熟 10-20%+ 利润率 | 亏损加速而无增长 |
| **40法则** | 收入增长% + 利润率% | 增长与效率的平衡 | >40 健康；25-40 可接受；<25 堪忧 | 40法则 <25 |
| **Magic Number** | (季度收入 - 上季度收入) × 4 / 上季度销售营销支出 | 销售营销效率 | >0.75 高效；0.5-0.75 可接受；<0.5 需修复GTM | Magic Number <0.5 |
| **运营杠杆** | 收入增长 vs. 运营支出增长 | 规模化效率 | 收入增长 > 运营支出增长 | 运营支出增速快于收入 |
| **毛收入 vs. 净收入** | 净收入 = 毛收入 - 折扣 - 退款 - 抵扣 | 你实际保留的 | 退款 <10%；折扣 <20% | 退款 >10%（产品问题） |
| **收入集中度** | 前N客户 / 总收入 | 对最大客户的依赖 | 最大客户 <10%；前10 <40% | 最大客户 >25%（存在风险） |
| **收入组合** | 产品/细分市场收入 / 总收入 | 产品组合构成 | 理想为无单一产品 >60% | 单一产品 >80%（无多元化） |
| **队列分析** | 按加入日期分组客户；追踪行为 | 业务在改善还是恶化 | 近期队列与旧队列持平或更好 | 新队列表现更差 |
| **按渠道CAC回收期** | CAC / 月度贡献（按渠道） | 按获客渠道的回收期 | 跨渠道比较 | 一个渠道远差于其他 |
| **毛利率回收期** | CAC / (月度ARPU × 毛利率%) | 使用实际利润的回收期 | 通常为简单回收期的1.5-2倍 | 使用利润率的回收期 >36个月 |
| **单位经济** | 每单位收入 - 每单位成本 | 每个"单位"的盈利能力 | 需要正的贡献 | 负的边际贡献率 |
| **细分市场回收期** | CAC / 月度贡献（按细分市场） | 按客户细分市场的回收期 | 比较以分配资源 | 一个细分市场不盈利 |
| **增量性** | 由行动引起的收入 - 基线 | 营销/促销的真实影响 | 用保留组衡量 | 庆祝无论如何都会发生的收入 |
| **营运资金** | 收入与收款之间的现金时间差 | 现金 vs. 收入时间差 | 年度预付 > 月度结算 | 长付款期扼杀跑道 |

### 快速决策框架

使用这些框架来组合指标用于常见PM决策。

#### 框架1：我们应该构建这个功能吗？

**问：**
1. **收入影响？** 直接（定价、附加功能）还是间接（留存、转化）？
2. **利润率影响？** COGS是多少？它会稀释利润吗？
3. **ROI？** 收入影响 / 开发成本

**构建的条件：**
- 第一年ROI >3倍（直接变现），或
- LTV影响 >10倍开发成本（留存），或
- 战略价值超越短期ROI

**不要构建的条件：**
- 即使乐观采用率下边际贡献率也为负
- 回收期超过平均客户生命周期

**需检查的指标：** 收入、毛利率、LTV、边际贡献率

---

#### 框架2：我们应该扩大这个获客渠道吗？

**问：**
1. **单位经济？** CAC、LTV、LTV:CAC比率
2. **现金效率？** 回收期
3. **客户质量？** 队列留存、NRR（按渠道）
4. **可扩展性？** Magic Number、可寻址体量

**扩大的条件：**
- LTV:CAC >3:1 且
- 回收期 <18个月 且
- 客户质量达到/超越其他渠道 且
- Magic Number >0.75

**不扩大的条件：**
- LTV:CAC <1.5:1 且
- 没有清晰的改善路径

**需检查的指标：** CAC、LTV、LTV:CAC、回收期、NRR、Magic Number

---

#### 框架3：我们应该改变定价吗？

**问：**
1. **ARPU/ARPA影响？** 每客户收入会增加吗？
2. **转化影响？** 帮助还是损害试用转付费转化率？
3. **流失影响？** 造成流失风险还是减少它？
4. **NRR影响？** 解锁扩展还是造成缩减？

**实施的条件：**
- 计入流失风险后净收入影响为正
- 可在广泛推出前在一个细分市场测试

**不改变的条件：**
- 高流失风险且无抵消性扩展
- 在承诺之前无法测试假设

**需检查的指标：** ARPU、ARPA、流失率、NRR、CAC回收期

---

#### 框架4：业务健康吗？

**按阶段检查：**

**早期阶段（Pre-¥10M ARR）：**
- 增长率 >50% 年同比
- LTV:CAC >3:1
- 毛利率 >70%
- 跑道 >12个月

**增长阶段（¥10M-¥50M ARR）：**
- 增长率 >40% 年同比
- NRR >100%
- 40法则 >40
- Magic Number >0.75

**规模化阶段（¥50M+ ARR）：**
- 增长率 >25% 年同比
- NRR >110%
- 40法则 >40
- 利润率 >10%

**需检查的指标：** 收入增长、NRR、LTV:CAC、40法则、Magic Number、毛利率

### 按类别的红旗指标

#### 收入与增长红旗
| **红旗** | **含义** | **行动** |
|--------------|-------------------|------------|
| 流失率逐队列增加 | 产品-市场匹配在恶化 | 停止扩大获客；先修复留存 |
| NRR <100% | 基础在缩减 | 在扩大之前先修复扩展或减少流失 |
| 收入流失 > 客户数流失 | 在流失大客户 | 调查为什么高价值客户离开 |
| Quick Ratio <2 | 漏水桶（勉强超过损失） | 在扩大获客前先修复留存 |
| 扩展收入 <MRR的10% | 没有向上销售/交叉销售引擎 | 构建扩展路径 |
| 收入集中度 >50%在前10客户 | 存在依赖风险 | 多元化客户基础 |

#### 单位经济红旗
| **红旗** | **含义** | **行动** |
|--------------|-------------------|------------|
| LTV:CAC <1.5:1 | 在亏损购买收入 | 扩大之前先降低CAC或提高LTV |
| 回收期 >24个月 | 现金陷阱（长现金回收周期） | 谈判年度预付或降低CAC |
| 毛利率 <60% | 每元收入的低盈利能力 | 提高价格或降低COGS |
| CAC上升而LTV持平 | 单位经济在恶化 | 优化转化或缩短销售周期 |
| 边际贡献率 <40% | 扣除可变成本后不盈利 | 削减可变成本或提高价格 |

#### 资本效率红旗
| **红旗** | **含义** | **行动** |
|--------------|-------------------|------------|
| 跑道 <6个月 | 生存危机 | 立即融资或削减烧钱 |
| 净烧钱加速而收入无增长 | 无结果下烧得更快 | 削减成本或增加收入紧迫性 |
| 运营支出增速快于收入 | 负运营杠杆 | 冻结招聘；优化支出 |
| 40法则 <25 | 无增长下烧钱 | 改善增长或削减至盈利 |
| Magic Number <0.5 | 销售营销引擎破裂 | 在扩大支出前先修复GTM效率 |

### 何时使用哪个指标

**功能优先级排序：**
- 收入影响 → 收入、ARPU、扩展收入
- 利润率影响 → 毛利率、边际贡献率
- ROI → LTV影响、开发成本

**渠道评估：**
- 获客成本 → CAC、按渠道CAC
- 客户价值 → LTV、按渠道NRR
- 回收期 → 回收期、按渠道CAC回收期
- 可扩展性 → Magic Number

**定价决策：**
- 变现 → ARPU、ARPA、ACV
- 影响 → 流失率、NRR、扩展收入
- 效率 → CAC回收期（定价变化会否影响它？）

**业务健康：**
- 增长 → 收入增长、MRR/ARR增长
- 留存 → 流失率、NRR、Quick Ratio
- 经济 → LTV:CAC、回收期、毛利率
- 效率 → 40法则、Magic Number、运营杠杆
- 生存 → 烧钱率、跑道

**董事会/投资者报告：**
- 关键指标：ARR、收入增长%、NRR、LTV:CAC、40法则、Magic Number、烧钱率、跑道
- 按阶段：早期阶段强调增长 + 单位经济；增长阶段强调40法则 + Magic Number；规模化阶段强调盈利能力 + 效率

## 示例

### 示例1：功能投资健全检查

你正在决定是否构建一个高级导出功能。

1. 使用框架1（我们应该构建这个功能吗？）
2. 提取基线指标：ARPU、毛利率、LTV、边际贡献率
3. 建模乐观、基准和下行采用率
4. 如果下行情景下边际贡献率为负，则拒绝

快速输出：
- 基准情景ROI：3.8倍
- 边际贡献率影响：+4个百分点
- 决定：立即构建，上线后90天检查流失和扩展情况

### 示例2：渠道扩大决策

付费社交带来了许多注册但留存薄弱。

1. 使用框架2（我们应该扩大这个获客渠道吗？）
2. 检查CAC、LTV:CAC、回收期和按渠道NRR
3. 与表现最好的渠道比较，而非公司平均水平

快速输出：
- LTV:CAC：1.6:1
- 回收期：26个月
- NRR：88%
- 决定：不扩大；限制支出并进行定向优化测试

## 常见陷阱

- 使用混合的公司平均值而非队列或渠道级别的指标
- 在Quick Ratio弱且留存恶化时扩大获客
- 认为高LTV:CAC就足够，而不检查回收期和跑道影响
- 仅基于ARPU提升就涨价，而不建模流失和缩减
- 在不匹配的公司阶段或商业模式之间比较基准
- 追踪很多指标却没有明确的决策问题
