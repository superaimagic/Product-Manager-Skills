## 目的

掌握收入和留存指标来理解SaaS商业势头、评估产品市场适配、做出数据驱动的增长投资决策。用于计算关键指标、解读趋势、及早发现问题以及向利益相关者传达商业健康状况。

这不是商业智能工具——它是PM理解哪些指标重要、如何正确计算以及基于数字该采取什么行动的框架。

## 核心概念

### 收入指标族

衡量业务产生多少收入的"顶线"指标。

**收入** — 销售产品/服务赚取的总金额，扣除费用前。利润表的"顶线"。
- **为什么PM关注：** 每个功能都应该连接到收入（直接或间接）。如果你无法阐述收入影响，优先级排序就变得不可能。
- **公式：** 期间内所有客户付款的总和
- **基准：** 增长率比绝对数字更重要（因阶段而异）

**ARPU（每用户平均收入）** — 每个独立用户产生的平均收入。
- **为什么PM关注：** 衡量按席位计费的有效性。对基于席位的定价模型至关重要。
- **公式：** `总收入 / 总用户数`
- **基准：** 因模式差异；追踪趋势比绝对值更重要
- **B2C SaaS：** ¥35-350/月典型；B2B：¥350-3500+/月

**ARPA（每账户平均收入）** — 每个客户账户产生的平均收入。
- **为什么PM关注：** 衡量账户级交易规模。对基于账户的定价模型至关重要。
- **公式：** `MRR / 活跃账户数`
- **基准：** 中小企业SaaS：¥700-¥7,000/月；中市场：¥7,000-¥70,000；企业：¥70,000+

**ARPA/ARPU分析** — 同时使用两个指标来理解变现。
- **为什么PM关注：** 防止包装错误。高ARPA + 低ARPU = 每席位变现不足。低ARPA + 高ARPU = 小交易规模。
- **示例：** ¥7万ARPA搭配100席位 = ¥700 ARPU（合理）。¥7万ARPA搭配1,000席位 = ¥70 ARPU（留下钱没赚）。

**ACV（年度合同价值）** — 每份合同的年化经常性收入（排除一次性费用）。
- **为什么PM关注：** 比较不同合同结构的经济效益。支持销售薪酬设计和细分分析。
- **公式：** `每份合同的年度经常性收入`（不含配置费、专业服务费）
- **基准：** 中小企业：¥3.5万-¥17.5万；中市场：¥17.5万-¥70万；企业：¥70万+

**MRR/ARR（月度/年度经常性收入）** — 标准化为月度或年度的可预测经常性收入。
- **为什么PM关注：** 订阅业务的心跳。估值5-10倍+乘数。追踪组成部分（新增、扩展、流失）。
- **公式：** `MRR = 所有月度订阅经常性收入之和`；`ARR = MRR × 12`
- **基准：** 增长率和质量重要；追踪新增MRR、扩展MRR、流失MRR、缩减MRR

**毛收入 vs. 净收入** — 折扣、退款、退款前的毛收入 vs. 之后的净收入。
- **为什么PM关注：** 折扣和退款可能隐藏差的获客质量或产品问题。
- **公式：** `净收入 = 毛收入 - 折扣 - 退款 - 退款`
- **基准：** 退款>10%是红旗；按获客渠道追踪

---

### 留存与扩展指标族

衡量你保留和增长现有客户的指标。

**流失率** — 期间内取消的客户百分比。
- **为什么PM关注：** SaaS的沉默杀手。侵蚀所有获客努力。5%月流失 = 46%年流失（复利效应）。
- **公式：** `期间内流失客户 / 期初客户数`
- **基准（月度）：** <2%优秀，2-5%可接受，>5%危机
- **基准（年度）：** <10%优秀，10-30%可接受，>30%危机
- **注意：** Logo流失（客户数）不同于收入流失（金额）

**NRR（净收入留存）** — 包括扩展和缩减的现有客户收入留存。
- **为什么PM关注：** 圣杯指标。NRR>100%意味着没有新Logo也能增长。投资者高度重视。
- **公式：** `(期初ARR + 扩展 - 流失 - 缩减) / 期初ARR × 100`
- **基准：** >120%优秀，100-120%良好，90-100%可接受，<90%有问题
- **示例：** 起始¥700万ARR，增加¥210万扩展，流失¥70万 = ¥840万 / ¥700万 = 120% NRR

**扩展收入** — 来自现有客户的额外收入（升级、交叉销售、用量增长）。
- **为什么PM关注：** 最高资本效率的收入（零CAC）。应驱动NRR>100%。
- **公式：** `现有客户的升级 + 交叉销售 + 用量增长之和`
- **基准：** 应占总收入20-30%；驱动NRR>100%

**快速比率（SaaS）** — 收入增益 vs. 收入损失。
- **为什么PM关注：** 显示你是在坚实基础上建设还是在跑步机上奔跑。
- **公式：** `(新增MRR + 扩展MRR) / (流失MRR + 缩减MRR)`
- **基准：** >4优秀，2-4健康，<2漏桶

---

### 分析框架

**收入组合分析** — 按产品、细分或渠道的收入分解。
- **为什么PM关注：** 识别哪些产品为业务提供资金以及投资方向。揭示集中风险。
- **公式：** `产品/细分收入 / 总收入 × 100`
- **基准：** 无单一产品>60%理想；多元化降低风险

**群组分析** — 按加入日期分组客户并随时间追踪行为。
- **为什么PM关注：** 混合指标隐藏关键趋势。显示业务是在改善还是恶化。
- **方法：** 按群组追踪留存、扩展和LTV（例如"2024年1月群组")
- **基准：** 近期群组应与老群组持平或更好

---

### 反模式（这不是什么）

- **不是利润指标：** 收入是顶线而非底线。高收入搭配负利润率是灾难。
- **不是虚荣指标：** 总收入增长如果由不可持续折扣或摧毁利润率的交易驱动就毫无意义。
- **不是混合平均值：** 将¥70中小企业和¥7,000企业客户平均的ARPU隐藏了细分经济效益。
- **不是孤立数字：** 流失率本身不讲述故事——需要看群组趋势和NRR。

---

### 何时使用这些指标

**在以下情况使用：**
- 评估整体商业健康和产品市场适配
- 比较跨时间段或群组的表现
- 优先有直接变现路径的功能（ARPU影响、扩展赋能）
- 与管理层、董事会或投资者沟通
- 评估留存问题（流失分析、群组退化）
- 衡量定价或包装变更（ARPU/ARPA变化）

**在以下情况不要使用：**
- 评估盈利能力（用毛利指标）
- 评估资本效率（用LTV:CAC、回本期）
- 没有成本上下文做产品投资决策（收入本身不是ROI）
- 没有标准化直接比较截然不同的商业模式

## 应用

### 步骤1：计算收入指标

使用 `template.md` 中的模板计算你的核心收入指标。

#### 收入
```
收入 = 期间内所有客户付款之和
```

**示例：**
- 月度1付款：¥700,000
- 收入 = ¥700,000

**质量检查：**
- 这是毛收入还是净收入？（明确折扣/退款是否包含）
- 收入是群组环比增长还是仅来自新客户增加？
- 收入增长率vs.人头/成本增长率是多少？

---

#### ARPU（每用户平均收入）
```
ARPU = 总收入 / 总用户数
```

**示例：**
- 总收入：¥700,000/月
- 总用户：2,000
- ARPU = ¥700,000 / 2,000 = ¥350/用户/月

**质量检查：**
- ARPU随时间增长还是缩减？
- ARPU增长来自涨价还是组合变化（流失小客户）？
- ARPU因群组差异吗？（新客户更/更有价值？）

---

#### ARPA（每账户平均收入）
```
ARPA = MRR / 活跃账户数
```

**示例：**
- MRR：¥700,000
- 活跃账户：200
- ARPA = ¥700,000 / 200 = ¥3,500/账户/月

**质量检查：**
- ARPA增长来自扩展还是仅更大新交易？
- ARPA跨客户细分如何比较？
- ARPA高但ARPU低吗？（每席位变现不足）

---

#### ARPA/ARPU联合分析
```
ARPA = MRR / 活跃账户数
ARPU = MRR / 总用户数
每账户平均席位 = ARPA / ARPU
```

**示例：**
- ARPA：¥3,500/月
- ARPU：¥350/月
- 平均席位：¥3,500 / ¥350 = 10席位/账户

**质量检查：**
- 你按席位变现有效吗？
- 你能每席位收费更多（提升ARPU）？
- 你能扩展每账户席位数（提升ARPA）？

---

#### ACV（年度合同价值）
```
ACV = 每份合同的年度经常性收入
（排除一次性费用如配置费、专业服务费）
```

**示例：**
- 客户签3年合同总额¥210万
- ACV = ¥210万 / 3年 = ¥70万/年

**质量检查：**
- ACV因细分差异吗（中小企业 vs. 企业）？
- ACV随时间增长吗（向上游走）？
- ACV能支撑销售团队成本结构吗？

---

#### MRR/ARR（月度/年度经常性收入）
```
MRR = 所有月度订阅经常性收入之和
ARR = MRR × 12

追踪组成部分：
- 新增MRR（来自新客户）
- 扩展MRR（来自升级/交叉销售）
- 流失MRR（来自流失客户）
- 缩减MRR（来自降级）
```

**示例：**
- 起始MRR：¥350万
- 新增MRR：+¥35万
- 扩展MRR：+¥14万
- 流失MRR：-¥10.5万
- 缩减MRR：-¥3.5万
- 结束MRR：¥385万
- ARR = ¥385万 × 12 = ¥4,620万

**质量检查：**
- MRR增长来自新客户还是扩展？
- 流失/缩减随增长增加吗？
- 新增:扩展:流失MRR比率是多少？（最佳：扩展 > 新增）

---

#### 毛收入 vs. 净收入
```
净收入 = 毛收入 - 折扣 - 退款 - 退款
```

**示例：**
- 毛收入：¥70万
- 折扣：-¥7万
- 退款：-¥1.4万
- 净收入：¥61.6万

**质量检查：**
- 折扣>20%吗？（定价权问题）
- 退款>10%吗？（产品质量问题）
- 某些渠道有更高折扣/退款率吗？

---

### 步骤2：计算留存与扩展指标

#### 流失率
```
Logo流失率 = 流失客户 / 期初客户数 × 100
收入流失率 = 流失MRR / 期初MRR × 100
```

**示例（Logo流失）：**
- 期初客户：1,000
- 流失客户：30
- Logo流失 = 30 / 1,000 = 3%月度

**示例（收入流失）：**
- 期初MRR：¥350万
- 流失MRR：¥10.5万
- 收入流失 = ¥10.5万 / ¥350万 = 3%月度

**质量检查：**
- 流失率随时间加速还是减速？
- 新群组流失比老群组更快吗？（PMF退化）
- 收入流失比Logo流失高吗？（流失大客户）

**月度转年度换算：**
- 月流失复利：3%月流失 ≠ 36%年流失
- 公式：`年流失 = 1 - (1 - 月流失)^12`
- 3%月流失 = ~31%年流失

---

#### NRR（净收入留存）
```
NRR = (期初ARR + 扩展 - 流失 - 缩减) / 期初ARR × 100
```

**示例：**
- 期初ARR：¥3500万
- 扩展：+¥560万
- 流失：-¥210万
- 缩减：-¥70万
- 群组期末ARR：¥3780万
- NRR = ¥3780万 / ¥3500万 = 108%

**质量检查：**
- NRR>100%吗？（没有新Logo也能增长）
- NRR群组环比改善还是恶化？
- 什么驱动NRR？（扩展还是低流失？）

---

#### 扩展收入
```
扩展收入 = 升级 + 交叉销售 + 用量增长（来自现有客户）
```

**示例：**
- 升级到更高层级：¥35万/月
- 交叉销售附加功能：¥14万/月
- 用量增长：¥7万/月
- 总扩展收入：¥56万/月

**质量检查：**
- 扩展收入占总收入比例增长吗？
- 每年多少客户扩展？（扩展率）
- 某些群组/细分更可能扩展吗？

---

#### 快速比率（SaaS）
```
快速比率 = (新增MRR + 扩展MRR) / (流失MRR + 缩减MRR)
```

**示例：**
- 新增MRR：¥35万
- 扩展MRR：¥14万
- 流失MRR：¥10.5万
- 缩减MRR：¥3.5万
- 快速比率 = (¥35万 + ¥14万) / (¥10.5万 + ¥3.5万) = ¥49万 / ¥14万 = 3.5

**质量检查：**
- 快速比率>4 = 优秀（增益远超损失）
- 快速比率2-4 = 健康（可持续增长）
- 快速比率<2 = 漏桶（先修留存再扩展）

---

### 步骤3：用框架分析趋势

#### 收入组合分析
```
产品/细分占比 = 产品/细分收入 / 总收入 × 100
```

**示例：**
- 产品A收入：¥210万
- 产品B收入：¥350万
- 产品C收入：¥140万
- 总收入：¥700万
- 产品A：30%，产品B：50%，产品C：20%

**质量检查：**
- 收入集中度增加吗？（风险：过度依赖一个产品）
- 哪些产品在增长/缩减？
- 收入组合匹配你的战略优先级吗？

---

#### 群组分析
按加入时间分组客户并随时间追踪指标。

**示例：**
| 群组 | 第0月 | 第1月 | 第2月 | 第3月 | 第6月 |
|------|-------|-------|-------|-------|-------|
| 2024年1月 | 100% | 95% | 92% | 90% | 85% |
| 2024年2月 | 100% | 94% | 90% | 87% | 80% |
| 2024年3月 | 100% | 92% | 86% | 82% | - |

**质量检查：**
- 近期群组留存比老群组好还是差？
- 如果更差：产品市场适配在退化（先修复再扩展）
- 如果更好：改进在生效（可以安全扩展）
- 追踪收入留存而非仅Logo留存

---

### 步骤4：质量检查与基准

在报告指标前，验证：

**收入指标：**
- ✅ 毛收入 vs. 净收入清晰标注
- ✅ 收入增长率 > 成本增长率
- ✅ ARPU/ARPA趋势按群组分析而非仅混合

**留存指标：**
- ✅ Logo流失和收入流失同时追踪
- ✅ 群组环比趋势分析而非仅混合流失
- ✅ NRR追踪组成部分（扩展、流失、缩减）

**分析：**
- ✅ 群组分析显示留存趋势
- ✅ 收入组合显示集中风险
- ✅ 快速比率显示增长可持续性

## 示例

查看 `examples/` 目录获取详细场景。迷你示例如下：

### 示例1：健康的SaaS指标

**公司：** 中市场项目管理SaaS

**收入指标：**
- MRR：¥1400万（月环比增长10%）
- ARR：¥1.68亿
- ARPA：¥8,400/月（200账户）
- ARPU：¥840/月（16,667用户）
- 平均席位：100每账户

**留存指标：**
- 月度Logo流失：2%
- 收入流失：1.5%（流失小客户）
- NRR：115%（强扩展）
- 扩展收入：¥140万/月（MRR的10%）
- 快速比率：5.0

**分析：**
- ✅ 强增长（10% MoM MRR）
- ️ 优秀留存（2% Logo流失，115% NRR）
- ️ 健康扩展（NRR>100%）
- ️ 可持续（快速比率5.0）
- ✅ 收入流失 < Logo流失（流失小客户，好信号）

**行动：** 扩大获客。单位经济效益强。

### 示例2：警告信号

**公司：** 中小企业营销自动化SaaS

**收入指标：**
- MRR：¥350万（月环比增长15%）
- ARR：¥4200万
- ARPA：¥1,750/月（2,000账户）
- ARPU：¥350/月（10,000用户）

**留存指标：**
- 月度Logo流失：6%（从6个月前4%增长）
- 收入流失：7%（流失更大客户）
- NRR：85%（缩减）
- 扩展收入：¥3.5万/月（MRR的1%）
- 快速比率：1.2

**群组分析：**
| 群组 | 第6月留存 |
|------|-----------|
| 6个月前 | 75% |
| 3个月前 | 65% |
| 当前 | 58% |

**分析：**
- ⚠️ 高流失（6%月度 ≈ ~50%年度）
- 🚨 收入流失 > Logo流失（流失大客户）
- 🚨 NRR<100%（缩减而非扩展）
- 🚨 群组退化（新客户流失更快）
- 🚨 快速比率1.2（漏桶）

**行动：** 停止扩大获客。先修留存。调查：
- 为什么新群组流失更快？
- 为什么扩展收入仅占MRR 1%？
- 什么导致客户缩减？

---

### 示例3：混合指标隐藏问题

**公司：** 多产品SaaS平台

**混合指标看起来很好：**
- MRR：¥2100万（月环比增长20%）
- 混合流失：3%
- 混合NRR：110%

**但收入组合分析显示：**
| 产品 | 收入 | 总收入占比 | 增长 | 流失 | NRR |
|------|------|-----------|------|------|-----|
| 遗留产品 | ¥1400万 | 67% | -5% MoM | 8% | 75% |
| 新产品 | ¥700万 | 33% | +80% MoM | 1% | 150% |

**分析：**
- 🚨 遗留产品（67%收入）在消亡：-5%增长，8%流失，75% NRR
- ✅ 新产品出色：+80%增长，1%流失，150% NRR
- ⚠️ 混合指标隐藏了2/3收入在缩减的事实
- ⚠️ 高度依赖一个产品（67%集中风险）

**行动：** 加速从遗留产品向新产品的迁移。规划遗留产品退出。

## 常见陷阱

### 陷阱1：混淆收入与利润
**症状：** "我们今年收入增长了50%，我们太棒了！"

**后果：** 收入是顶线而非底线。你可能以亏损增长、摧毁利润率或扩展不盈利产品。

**修复：** 总是将收入指标搭配毛利指标（见 `saas-economics-efficiency-metrics`)。¥100万收入80%毛利 >> ¥200万收入20%毛利。

---

### 陷阱2：庆祝来自组合变化的ARPU增长
**症状：** "ARPU增长30%！"（但客户数下降了40%）

**后果：** ARPU上升是因为你失去了所有小客户，而非你改善了变现。

**修复：** 按群组和细分分析ARPU。真正的ARPU改善 = 同样客户付更多，而非流失便宜客户。

---

### 陷阱3：忽略群组退化
**症状：** "混合流失稳定在3%"

**后果：** 混合指标可能隐藏新群组流失6%而老群组流失1%。产品市场适配在退化。

**修复：** 总是按群组分析留存。如果近期群组表现更差，停止扩展并修复产品。

---

### 陷阱4：Logo流失 vs. 收入流失混淆
**症状：** "Logo流失只有2%，我们很好！"

**后果：** 你可能流失2%客户但10%收入，如果你在流失大客户。

**修复：** 同时追踪Logo流失和收入流失。如果收入流失 > Logo流失，你在流失高价值客户。

---

### 陷阱5：等同对待所有流失
**症状：** "这个月我们流失了50个客户"（没有关于谁的上下文）

**后果：** 流失50个小客户（¥70/月）与流失50个企业客户（¥7万/月）不同。

**修复：** 按客户大小、群组和原因细分流失分析。按收入影响而非仅Logo数加权。

---

### 陷阱6：忘记复利流失
**症状：** "3%月流失还好，那只是36%年流失"

**后果：** 流失复利。3%月流失 = 31%年流失，不是36%。数学：`1 - (1 - 0.03)^12 = 31%`。

**修复：** 月转年时使用正确公式。不要简单乘12。

---

### 陷阱7：庆祝毛收入但净收入缩减
**症状：** "毛收入增长20%！"（但折扣/退款翻倍）

**后果：** 净收入可能持平或缩减。折扣隐藏定价权问题；退款隐藏产品质量问题。

**修复：** 总是追踪毛收入和净收入。如果折扣>20%或退款>10%，调查原因。

---

### 陷阱8：NRR>100%来自低流失而非扩展
**症状：** "NRR是105%，我们在扩展！"

**后果：** NRR可以仅从非常低流失就>100%，没有有意义的扩展。真正扩展驱动的NRR是>120%。

**修复：** 将NRR分解为组成部分：扩展MRR vs. 流失/缩减MRR。目标扩展驱动的NRR而非仅低流失。

---

### 陷阱9：收入集中风险
**症状：** "我们达到¥7000万ARR了！"（但¥3500万来自一个客户）

**后果：** 失去那一个客户收入减半。路线图成为那一个客户需求的囚徒。

**修复：** 追踪收入集中度。理想：Top客户<收入10%，Top 10客户<收入40%。尽早多元化。

---

### 陷阱10：跨细分平均ARPU/ARPA
**症状：** "我们的ARPU是¥700"（¥70中小企业和¥7,000企业客户的平均）

**后果：** 混合ARPU隐藏细分经济效益。无法做出聪明的获客或产品决策。

**修复：** 按细分（中小企业、中市场、企业）计算ARPU/ARPA。独立优化每个细分。
