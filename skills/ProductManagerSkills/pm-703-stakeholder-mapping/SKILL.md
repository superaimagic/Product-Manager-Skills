# 利益相关者映射

## 目的

对利益相关者进行优先排序并设定参与策略。这项技能在你已识别的利益相关者集合上运行两个互补网格，并有意比较它们揭示的内容，因为每个网格展示的是另一个无法展示的东西。

**权力×兴趣**回答：每位利益相关者对这个项目的关心程度如何，他们能多大程度地影响它？它产出参与策略——谁需要深度参与、谁需要保持知情、谁需要关注监控。这是大多数产品经理熟悉的网格。

**影响×权力**回答：谁承受这个产品结果的后果，他们实际拥有多少组织权力？它浮现最需要做好但最可能被低估的利益相关者——高影响、低权力。

只运行第一个网格会优化与权力者的关系管理。只运行第二个会产出公平性分析但没有参与策略。洞见存在于比较之中：一个在第一个网格中"保持知情"、在第二个网格中"高影响、低权力"的利益相关者，是你一直在低估参与度的对象——而后果完全落在他们身上。这是产品风险。

在stakeholder-identification（构建完整列表）之后、stakeholder-engagement-advisor（规划逐人参与）之前使用本技能。

## 核心概念

**权力×兴趣网格** — 一个2×2矩阵，按利益相关者影响项目的能力（纵轴）和对结果的兴趣程度（横轴）进行定位。产出四个象限及相应的参与策略：
- 高权力、高兴趣 → **密切管理**（共同设计、频繁沟通）
- 高权力、低兴趣 → **保持满意**（高管简报、战略框架）
- 低权力、高兴趣 → **保持知情**（通讯、演示、透明度）
- 低权力、低兴趣 → **关注监控**（轻接触、仅在需要时沟通）

**影响×权力网格** — 一个2×2矩阵，按利益相关者是否显著受项目影响（纵轴）和其拥有的权力（横轴）进行定位。产出四个象限：
- 高影响、高权力 → **Q2** — 受影响且有权力；密切管理
- 高影响、低权力 → **Q1** — 受影响但边缘化；有意识地提升其声音
- 低影响、高权力 → **Q4** — 把关人；管理关系，降低参与深度
- 低影响、低权力 → **Q3** — 关注监控，最小投入

**提升Q1声音** — 高影响、低权力利益相关者（Q1）是最可能经历产品失败模式、最不可能出现在你常规反馈循环中的人。有意识地增加他们在路线图决策中的角色——通过研究招募、可用性测试和需求评审——不仅是公平的，还能降低构建错误问题的产品风险。

**象限迁移** — 通过有针对性的行动将利益相关者从当前象限迁移到期望象限的策略。一个目前在"关注监控"中的怀疑高管需要成为赞助方，所需的一套行动与已在"密切管理"中的不同。将迁移显化使地图从快照变为计划。

**影响与权力的区别** — 这不是同一维度。一线客服人员影响很大（日常工作流完全改变）但权力很低（没有路线图决策席位）。财务副总裁权力很大（预算审批）但影响很低（产品不改变其工作方式）。混淆两者是最常见的映射错误。

**配对变量** — 运行两个网格而非一个的原因。不同的配对轴揭示不同的关系。单一网格产出一幅单一的、不完整的画面。运行多个分析并比较输出才是真正洞见所在。

**参与策略** — 每个象限分配的差异化沟通和参与方式。它将有限的产品经理带宽分配到杠杆最高的地方——并防止对所有利益相关者一视同仁的陷阱，那意味着在低 stakes 关系上投入而忽视关键关系。

## 应用方法

**步骤1 — 运行权力×兴趣网格**

从识别练习中对每位利益相关者评估：
- **权力：** 此人或群体能否显著塑造、监管、审批或阻止本项目？（考虑正式权威、预算、否决权和非正式影响力。）
- **兴趣：** 此人或群体是否积极参与或受结果影响？他们是否有已表达或可能的关于方向的意见？

将每位利益相关者放入相应象限。此步骤不需要在象限内排序或量化——定位本身就是输出。

为每个象限分配参与策略：
- 密切管理 → 深度参与、共同设计会议、每周或双周沟通
- 保持满意 → 高管简报、高层框架、里程碑更新
- 保持知情 → 结构化沟通、演示、工件只读访问
- 关注监控 → 轻接触沟通、可用但非主动追踪

**步骤2 — 运行影响×权力网格**

对每位利益相关者分别评估：
- **影响：** 本项目是否显著影响此人或群体——财务上、职业上、日常工作中或服务获取方面？如果是，则为高影响。
- **权力：** 他们能否显著塑造本项目，无论他们是否愿意？（定义同步骤1，但独立于兴趣评估。）

将每位利益相关者放入Q1-Q4象限。在不参考权力×兴趣定位的情况下进行——两项评估的独立性正是比较价值所在。

**步骤3 — 比较并发现缺口**

将两个网格并排放置并问：

- 谁在网格1中是"保持知情"、在网格2中是Q1（高影响、低权力）？这些是你参与度不足但产品对他们影响最大的利益相关者。
- 谁在网格1中是"密切管理"、在网格2中是Q3/Q4？这些是权力很大但利益关联低的利益相关者——注意过度投入。
- 谁在两个网格间有显著位移？这些位移揭示了权力与后果错位的地方。

**步骤4 — 规划象限迁移**

对于你想迁移的利益相关者：
- Q1 → Q2：积极将其纳入顾问流程、探索性研究和需求评审。给他们明确的角色，而非仅是被动的访问权。
- 关注监控 → 保持知情：增加透明度以建立意识和早期对齐，在项目对他们变得相关之前。
- 任何象限中的怀疑者 → 盟友：识别他们需要相信什么才能支持，然后设计一对一参与来直接解决这些关切。

记录：谁在迁移、什么行动触发迁移、谁负责执行。

**步骤5 — 输入参与规划**

为每位"密切管理"象限和每位Q1利益相关者，使用stakeholder-engagement-advisor创建参与计划。这些关系是战术规划——具体信息、媒介、节奏和成功标准——回报最高的地方。

## 示例

**情境：** 一个平台团队正在迁移内部API基础设施。利益相关者列表：工程副总裁（赞助方）、三位工程负责人（直接用户）、法务（合规审查）、客服团队（其工具依赖这些API）、以及不会直接看到变更但其可用性依赖此的企业客户。

**权力×兴趣网格定位：**
- 工程副总裁 → 密切管理（高权力、高兴趣）
- 工程负责人 → 密切管理（高权力、高兴趣 — 直接建设者）
- 法务 → 保持满意（高权力、低日常兴趣）
- 客服 → 保持知情（低权力、高兴趣 — 工具会变）
- 企业客户 → 关注监控（低权力、低表达兴趣）

**影响×权力网格定位：**
- 工程副总裁 → Q2（高影响、高权力）
- 工程负责人 → Q2（高影响、高权力）
- 法务 → Q4（低影响、高权力）
- 客服 → **Q1**（高影响、低权力 — 如果迁移失败其工具会崩溃）
- 企业客户 → **Q1**（高影响、低权力 — 可用性依赖）

**比较揭示的缺口：** 客服和企业客户在网格1中是"关注监控"或"保持知情"，但在网格2中是Q1。迁移团队一直在将他们当作被动观察者，但他们实际上是最高风险的利益相关者。解决方案：将客服人员纳入UAT、创建包含回滚触发条件的企业客户沟通计划，并将两组都加入上线就绪标准。

## 常见误区

**只运行一个网格。** 权力×兴趣没有影响×权力会优化与权力者的关系管理而系统性亏待高影响、低权力群体。影响×权力没有权力×兴趣产出没有参与策略的公平性分析。两个网格都有其价值。

**混淆组织层级与权力。** 一个控制你项目审批队列的中层项目经理比不参与的高管拥有更多实际权力。非正式影响力——流程知识、把关、联盟建设——是真实的权力。评估它，不要假设组织架构图反映了它。

**将定位当作永久状态。** 组织变动、预算周期和利益相关者轮换会将人移到不同象限。建立季度评审周期或在重大组织事件后重新运行的节奏。

**停在定位而没有参与行动。** 两个带点的网格和没有"谁接下来做什么"就是一张漂亮的幻灯片，而非计划。地图是迁移策略和逐人参与计划的手段。

**在定位时量化或在象限内排序。** 方法明确反对这样做。分类步骤中的虚假精度会将讨论引向争论相对排名而非识别谁缺失于正确象限。

**让Q1利益相关者仅停留在命名。** 在画布上命名高影响、低权力利益相关者然后继续以同样方式参与他们是最常见的失败。提升需要具体行动：一个命名的调研名额、一个共同设计会议、一个需求评审角色、一个上线就绪关卡的参与席位。
