# 扣子技能商店打包上传指南

## 一、打包前检查清单

### 必需文件
- ✅ `SKILL.md` - 技能定义主文件
- ✅ `manifest.json` - 技能元数据配置
- ✅ `assets/` - 可复用资产目录
- ✅ `references/` - 参考资料目录
- ✅ `examples/` - 示例目录

### 文件结构
```
user-story/
├── SKILL.md
├── manifest.json
├── assets/
│   ├── user-story-template.md
│   └── acceptance-criteria-checklist.md
├── references/
│   └── framework-docs.md
└── examples/
    └── sample.md
```

## 二、打包步骤

### Windows系统
1. 进入 `g:\WWW\GitHub\Product-Manager-Skills\skills\` 目录
2. 右键点击 `user-story` 文件夹
3. 选择"发送到" → "压缩(zipped)文件夹"
4. 得到 `user-story.zip`

### PowerShell命令行
```powershell
cd g:\WWW\GitHub\Product-Manager-Skills\skills\
Compress-Archive -Path user-story -DestinationPath user-story.zip
```

## 三、扣子网页端上传

### 步骤1：登录扣子平台
- 访问：https://www.coze.cn/
- 登录账号（需要实名认证）

### 步骤2：进入技能商店
- 点击右上角"创建" → "创建技能"
- 或访问：https://www.coze.cn/skills

### 步骤3：上传技能包
- 点击"上传技能包"
- 选择 `user-story.zip` 文件
- 等待系统解析

### 步骤4：填写技能信息
| 字段 | 内容 |
|------|------|
| **技能名称** | 用户故事生成器 |
| **技能描述** | 输入一句话需求，自动生成开发就绪的用户故事（三段式格式 + 验收标准三步法）。PM写需求、PO拆分故事、团队统一标准。 |
| **分类** | 效率工具 / 产品管理 |
| **标签** | 用户故事, 需求管理, 敏捷开发, PM工具 |
| **定价** | 免费（建议先免费引流） |

### 步骤5：提交审核
- 检查所有信息无误
- 点击"提交审核"
- 等待平台审核（通常1-3个工作日）

## 四、审核要求

### 必须满足
- ✅ 开发者实名认证（个人或企业）
- ✅ 技能描述具体，避免"万能""超级"等夸大词汇
- ✅ 功能完整，可正常使用
- ✅ 提供3个以上可访问的测试案例链接

### 测试案例准备
需要准备3个对话链接，展示技能的不同使用场景：

**案例1：基础用户故事生成**
```
用户输入：帮我写一个微信登录的用户故事
技能输出：完整的用户故事（三段式 + 验收标准）
```

**案例2：PRD需求转化**
```
用户输入：把这段需求转成用户故事："用户需要在个人中心修改手机号"
技能输出：格式正确的用户故事
```

**案例3：故事质量检查**
```
用户输入：检查这条用户故事有没有问题："作为用户，我想要更好的登录体验"
技能输出：指出问题并给出改进建议
```

## 五、常见问题

### Q1: 上传失败怎么办？
- 检查zip文件大小（建议<5MB）
- 确认manifest.json格式正确
- 确保SKILL.md编码为UTF-8

### Q2: 审核被拒怎么办？
- 查看审核反馈意见
- 修改技能描述或功能
- 补充测试案例链接
- 重新提交审核

### Q3: 如何更新已上架技能？
- 在技能管理页面点击"编辑"
- 上传新的zip包
- 提交审核
- 审核通过后自动更新

## 六、Bot商店 vs 技能商店

如果技能商店上传不成功，可以尝试Bot商店方式：

### Bot商店流程
1. 创建新Bot
2. 复制 `coze/bot-config.md` 中的人设与回复逻辑
3. 配置开场白和变量
4. 发布到Bot商店

### 两种方式对比
| 方式 | 优势 | 劣势 |
|------|------|------|
| **技能商店** | 更专业、可收费、独立展示 | 审核严格、需要案例链接 |
| **Bot商店** | 审核宽松、快速上架 | 对话型、不适合工具技能 |
