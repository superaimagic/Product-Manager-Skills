## 一、扣子平台 Skill 架构分析

### 1.1 核心理念

根据扣子官方文档和实战教程，扣子 Skill 的核心理念是：

> **"Skill 是可沉淀的做事方法，把每次要重新对 AI 说的话变成预设的指引，把不稳定的生成变成可预期的结果。"**

这意味着 Skill 不仅仅是 Prompt，而是一套完整的方法论固化。

### 1.2 Skill 结构规范

扣子 Skill 的标准结构包括：

```
Skill 资产结构：
├── SKILL.md（流程指引）
├── Assets（设计相关参考资产）
│   ├── layout-guide.md（设计规范）
│   └── html-template.html（可复用模板）
└── reference（参考资料）
```

**关键特点**：
- **SKILL.md**：核心流程指引，定义 Skill 的做事方法
- **Assets**：可复用的资产文件（模板、规范、样式等）
- **reference**：参考资料，提供理论支撑

### 1.3 与 Claude Code Skill 的对比

| 维度 | 扣子 Skill | Claude Code Skill |
|------|------------|-------------------|
| **运行环境** | 云端 | 本地 |
| **安装复杂度** | 无需安装，开箱即用 | 安装较复杂 |
| **文件读写** | 不能实时读写本地文件 | 可实时读写电脑文件 |
| **自动化任务** | 友好（无需保持电脑开机） | 电脑需保持开机、亮屏 |
| **分发能力** | 可直接分享，团队使用方便 | 较难分发 |
| **适用场景** | 不需要本地文件读写的任务 | 需要代理操作浏览器的复杂行动 |

### 1.4 扣子技能商店生态

**当前商店 Skills 分类**：
- 自动化创作类：自动化写作、卖货文案生成、TED 视频转公众号文章
- 可视化生成类：互动教学课件、文章转 PPT、古诗插画生成
- 设计开发类：3D 图标生成、前端设计强化、公众号 SVG 生成器
- 专业领域知识类：投资机构观点、考研英语强化、商业分析 AI 助理
- 实用工具类：法律类案检索、SEO 优化、arXiv 论文搜索、文本转 EPUB

**定价模式**：
- 免费：完全免费供他人使用
- 按月付费：周期性收费
- 需要开通商户账户才能收费

**重要限制**：
- 个人版用户无法上架技能商店
- 只有企业版可以公开发布技能
- 需要提交资料并通过审核

## 二、当前 Product-Manager-Skills 项目分析

### 2.1 项目概况

**基本信息**：
- **项目名称**: Product Manager Skills
- **当前版本**: v0.85-CN（原版 v0.80）
- **技能总数**: 68 个（原始 52 个 + 中文本土化 16 个）
- **开源协议**: CC BY-NC-SA 4.0（非商业用途）
- **GitHub Stars**: 活跃开源项目

**技能分类**（按类型）：
- **Component Skills (25 个)**：模板类技能，生成特定 PM 交付物（30-90 分钟）
- **Interactive Skills (27 个)**：引导式发现，通过 3-5 个问题提供推荐
- **Workflow Skills (8 个)**：完整的端到端 PM 流程（数天到数周）

**中文本土化进度**：
- Component: 6 个中文版（user-story-cn, jobs-to-be-done-cn, problem-statement-cn, customer-journey-map-cn, agile-sprint-planning, chinese-user-research）
- Interactive: 5 个中文版（prioritization-advisor-cn, opportunity-solution-tree-cn, ab-testing-advisor, product-growth-hacking, product-monetization-advisor）
- Workflow: 5 个中文版（prd-development-cn, discovery-process-cn, roadmap-planning-cn, product-strategy-session-cn, okr-management）

### 2.2 现有打包策略

项目已经实现了按功能拆分的打包策略：

| 包名 | 下载链接 | 内容 |
|------|---------|------|
| **Starter Pack** | `pm-skills-starter-pack.zip` | 所有类别的核心技能 |
| **Discovery Pack** | `02-discovery-pack.zip` | 研究、访谈、综合分析 |
| **Strategy Pack** | `03-strategy-pack.zip` | 定位、路线图、优先级排序 |
| **Delivery Pack** | `04-delivery-pack.zip` | PRD、用户故事、Epic |
| **AI PM Pack** | `05-ai-pm-pack.zip` | 上下文工程、编排、准备度 |
| **All Skills Pack** | `99-all-skills-pack.zip` | 所有 49 个技能（原版） |

**支持平台**：
- Claude Desktop / Web
- Claude Code（插件市场）
- Codex
- ChatGPT（自定义 GPT 知识库）
- Cursor、Windsurf、n8n、LangFlow、CrewAI、Gemini 等

### 2.3 项目优势

1. **完整性**：覆盖 PM 工作的全流程，从发现问题到交付产品
2. **教学性**：遵循 ABC（Always Be Coaching）原则，不仅生成交付物，还传授方法论
3. **实战性**：所有技能都包含真实案例、反模式（Common Pitfalls）、参考资料
4. **多平台支持**：不仅支持 Claude 系列，还支持多种 AI Agent 平台
5. **开源社区**：活跃的 GitHub 社区，持续更新和维护

### 2.4 项目不足（针对扣子平台适配）

1. **粒度过大**：当前技能包是按功能分类（如 Discovery Pack），但每个 Pack 内仍包含多个技能，不符合扣子"一个 Skill 解决一个具体问题"的理念
2. **缺乏 Assets 和 reference**：当前技能只有 SKILL.md，没有扣子推荐的 Assets（可复用模板）和 reference（参考资料）目录
3. **云端适配问题**：当前技能设计为本地运行（可读写文件），但扣子平台是云端运行，无法读写本地文件
4. **缺乏商业化设计**：当前是开源免费，没有考虑扣子技能商店的付费模式
5. **中文适配不够深入**：虽然有 16 个中文版技能，但只是翻译，没有针对中国 PM 的实际场景做深度优化（如微信生态、私域运营、下沉市场等）

## 三、完整 vs 拆分：利弊分析

### 3.1 方案 A：保持完整（一个大全包）

**定义**：将所有 68 个技能打包成一个完整的 Product-Manager-Skills 包，用户一次性获取所有技能。

**优势**：
1. **完整性**：用户可以获得完整的 PM 工具集，无需多次下载
2. **一致性**：所有技能使用相同的术语、格式、方法论，学习曲线平滑
3. **品牌价值**："完整的 PM 技能库"是一个强有力的价值主张
4. **维护简单**：只需维护一个包，更新和发布流程简单

**劣势**：
1. **体积过大**：68 个技能打包后可能很大，影响下载和加载速度
2. **学习曲线陡峭**：新手面对 68 个技能可能不知从何下手
3. **不符合扣子理念**：扣子推荐"一个 Skill 解决一个具体问题"，完整包过于庞大
4. **定价困难**：免费吧，投入产出比低；付费吧，价格定高了用户不愿意买，定低了不值得
5. **更新风险**：一个技能更新，整个包需要重新发布，用户需要重新下载

**适用性**：
- ✅ 适合已经在用 Claude Code 的 PM（他们习惯一次性安装所有技能）
- ✅ 适合企业内部分发（一次性分发给所有 PM）
- ❌ 不适合扣子技能商店（用户更倾向于按需下载）

### 3.2 方案 B：按功能拆分（推荐）

**定义**：将 68 个技能按功能进一步拆分，每个 Skill 解决一个具体问题的完整方法论。

**拆分粒度建议**（参考扣子平台的成功案例）：

#### 第一层拆分：按使用场景（已在做）
- Discovery Pack（发现与调研）
- Strategy Pack（策略与规划）
- Delivery Pack（交付与执行）
- AI PM Pack（AI 产品管理）
- Leadership Pack（领导力与职业发展的技能，目前缺失）

#### 第二层拆分：每个 Pack 内的技能独立成包（扣子推荐）
以 Discovery Pack 为例，可以拆分为：
1. **用户访谈准备 Skill**：只包含 `discovery-interview-prep` 技能
2. **机会解决方案树 Skill**：只包含 `opportunity-solution-tree` 技能
3. **发现流程引导 Skill**：只包含 `discovery-process` 技能
4. **客户旅程地图 Skill**：只包含 `customer-journey-map` 技能
5. **Jobs-to-be-Done 分析 Skill**：只包含 `jobs-to-be-done` 技能

**优势**：
1. **符合扣子理念**：每个 Skill 解决一个具体问题，用户按需下载
2. **降低决策成本**：用户不需要理解 68 个技能，只需找到当前需要的那一个
3. **灵活定价**：可以为高价值 Skill 单独定价（如 PRD 开发、战略规划）
4. **易于维护**：单个 Skill 更新不影响其他 Skill
5. **提高转化率**：免费 Skill 可以引流，付费 Skill 实现变现

**劣势**：
1. **维护成本增加**：需要维护 68 个独立的 Skill 包（如果完全拆分）
2. **用户体验碎片化**：用户可能需要下载多个 Skill 才能完成一个完整流程
3. **品牌价值稀释**：用户可能只看到一个 Skill，不知道这是完整工具集的一部分

**解决方案**：
- **折中方案**：先按功能拆分为 5-6 个 Pack（第一层拆分），每个 Pack 作为一个独立产品在扣子商店上架
- **未来迭代**：根据用户在扣子平台的反馈，再决定是否进一步拆分为单个 Skill

**适用性**：
- ✅ 适合扣子技能商店（用户习惯按需下载）
- ✅ 适合免费+付费混合模式（基础 Skill 免费，高级 Skill 付费）
- ✅ 适合中国市场（中国 PM 更喜欢"即用即走"的工具）
- ⚠️ 需要额外的适配工作（云端化、Assets 和 reference 补充）

### 3.3 方案 C：混合模式（最佳实践）

**定义**：结合完整包和拆分包的优势，提供多种获取方式。

**具体策略**：
1. **扣子技能商店**：按功能拆分为 5-6 个 Pack，每个 Pack 独立上架
2. **GitHub Releases**：保持完整的 `all-skills-pack.zip` 供高级用户下载
3. **Claude Code 插件市场**：保持完整的插件（一次性安装所有技能）
4. **官方网站/文档**：提供"Skill 浏览器"，用户可以在线预览所有 Skill，然后选择下载单个 Pack 或完整包

**优势**：
1. **覆盖多平台**：不同平台的用户可以选择最适合自己的获取方式
2. **最大化触达**：扣子商店获取新用户，GitHub 服务开发者，Claude Code 服务深度用户
3. **灵活定价**：扣子商店可以按 Pack 收费，GitHub 可以接受捐赠或企业授权
4. **降低维护成本**：核心技能库统一维护，只是打包方式不同

**实施步骤**：
1. 先完成扣子平台的适配（云端化、Assets 和 reference 补充）
2. 按功能拆分为 5-6 个 Pack，上架扣子商店
3. 保持 GitHub 的完整包下载
4. 建立官方网站，提供 Skill 浏览器和文档

## 四、定价策略建议

### 4.1 扣子技能商店定价模式

根据扣子官方文档，支持以下定价模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **免费** | 完全免费供他人使用 | 引流款、基础功能 |
| **按月付费** | 周期性收费 | 高级功能、持续更新 |

**付费设置流程**：
1. 在「开通商户」入口开通收款账户
2. 根据指引提交资料
3. 等待审核通过

### 4.2 推荐定价策略：免费+付费混合模式

#### 策略一：基础免费，高级付费

**免费 Skill（引流款）**：
- `user-story-cn`：用户故事撰写（最常用）
- `problem-statement-cn`：问题陈述（入门必备）
- `jobs-to-be-done-cn`：JTBD 分析（基础框架）

**目的**：降低用户尝试门槛，建立信任，引流到付费 Skill

**付费 Skill（利润款）**：
- `prd-development-cn`：PRD 开发全流程（高价，因为输出是直接可用的文档）
- `product-strategy-session-cn`：产品策略会议引导（高价，因为涉及多利益相关者对齐）
- `roadmap-planning-cn`：路线图规划（中高价，因为输出是战略级文档）

**定价建议**：
- 单次购买：¥9.9 - ¥29.9（根据复杂度和输出价值）
- 按月订阅：¥19.9/月（包含所有付费 Skill 的使用权）

#### 策略二：按 Pack 定价

**Starter Pack（免费）**：
- 包含 3 个基础 Skill（user-story-cn, problem-statement-cn, jobs-to-be-done-cn）
- 目的：让用户体验产品质量

**Professional Pack（¥49.9）**：
- 包含 10 个核心 Skill（PRD、优先级排序、路线图、用户故事、验收标准等）
- 目的：服务职业 PM

**Enterprise Pack（¥99.9 或按需定价）**：
- 包含全部 68 个技能
- 额外提供：企业级支持、定制化、团队培训
- 目的：服务企业和团队

#### 策略三：免费+捐赠（开源精神）

**所有 Skill 免费**，但提供捐赠入口：
- 用户可以使用所有 Skill
- 如果觉得有用，可以自愿捐赠
- 捐赠用户获得：优先支持、新 Skill 内测资格、定制化需求投票权

**优势**：
- 符合开源协议（CC BY-NC-SA 4.0 允许免费分发）
- 快速积累用户和口碑
- 避免定价和商户审核的麻烦

**劣势**：
- 无法直接变现
- 需要其他盈利模式（如企业咨询、培训、定制化开发）

### 4.3 推荐方案：策略一 + 策略三结合

**具体做法**：
1. **基础 Skill 免费**（如 user-story-cn, problem-statement-cn）
2. **高级 Skill 付费**（如 prd-development-cn, product-strategy-session-cn）
3. **完整包捐赠**（所有 Skill 打包，捐赠后即可获取）
4. **企业服务单独定价**（企业内部分发、定制化、培训等）

**定价细节**：
- 免费 Skill：3-5 个，覆盖最常用的基础场景
- 付费 Skill：¥9.9 - ¥29.9 单次购买，或 ¥19.9/月订阅
- 完整包：捐赠 ¥49.9 以上即可获取所有 Skill
- 企业服务：按需定价（通常 ¥5000+ / 项目）

## 五、扣子平台适配行动计划

### 5.1 第一阶段：技能云端化适配（1-2 周）

**目标**：让现有技能可以在扣子平台运行（云端环境，无法读写本地文件）

**任务清单**：
1. **识别文件操作**：找出所有涉及本地文件读写的技能
   - 例如：`prd-development` 可能会生成 `.md` 文件保存到本地
   - 例如：`customer-journey-map` 可能会生成可视化图表保存到本地

2. **改造为云端输出**：将文件输出改为直接在对话中展示
   - 例如：PRD 直接在对话中生成 Markdown 文本，用户手动复制
   - 例如：客户旅程地图生成 Mermaid 代码，用户在扣子中渲染

3. **补充 Assets 和 reference**：按照扣子规范，为每个技能添加可复用资产
   - 例如：`prd-development` 的 Assets 中可以包含 PRD 模板（Markdown）
   - 例如：`user-story` 的 Assets 中可以包含用户故事模板、验收标准模板

**交付物**：
- 10 个核心技能完成云端化适配
- 每个技能都包含 SKILL.md + Assets + reference

### 5.2 第二阶段：中文本土化深度优化（1-2 周）

**目标**：不仅翻译，还要针对中国 PM 的实际场景做深度优化

**优化方向**：
1. **案例本土化**：使用中国产品的案例（如微信、抖音、淘宝、美团）
2. **方法论适配**：融入中国互联网的独特方法论（如增长黑客、私域运营、裂变增长）
3. **术语本地化**：使用中国 PM 熟悉的术语（如"需求文档"而非"PRD"，"迭代"而非"Sprint"）
4. **工具推荐**：推荐中国 PM 常用的工具（如飞书、钉钉、腾讯文档、ProcessOn）

**具体任务**：
- 优化已完成的 16 个中文版技能
- 新增 10 个中国场景专属技能（如 `wechat-ecosystem-research`, `private-domain-operations`, `aarrr-metrics-cn`）

**交付物**：
- 26 个深度中文本土化技能
- 10 个中国场景专属技能

### 5.3 第三阶段：扣子技能商店上架（1 周）

**目标**：将技能上架到扣子技能商店，开始获取用户和变现

**任务清单**：
1. **注册企业版扣子**：个人版无法上架商店，需要注册企业版
2. **准备上架素材**：
   - 技能名称（简洁、易搜索）
   - 技能简介（一句话说清楚价值）
   - 封面图（专业、吸引眼球）
   - 精选案例（展示技能的实际效果）
   - 使用文档（帮助用户快速上手）

3. **按 Pack 上架**：
   - Starter Pack（免费）
   - Professional Pack（¥49.9）
   - 单个高级 Skill（¥9.9 - ¥29.9）

4. **设置付费模式**：
   - 开通商户账户
   - 提交资料并通过审核
   - 设置免费+付费混合模式

**交付物**：
- 3-5 个 Pack 上架扣子技能商店
- 付费模式配置完成

### 5.4 第四阶段：持续运营与迭代（长期）

**目标**：根据用户反馈持续优化技能，扩大用户基础

**运营策略**：
1. **收集用户反馈**：在扣子中设置反馈入口，了解用户需求
2. **快速迭代**：根据用户反馈，每周更新 1-2 个技能
3. **内容营销**：在知乎、微信公众号、掘金等平台发布 PM 方法论文章，引流到扣子商店
4. **社区建设**：建立微信社群，聚集 PM 用户，提供免费咨询和技能试用

**增长策略**：
1. **免费 Skill 引流**：让用户先试用免费 Skill，体验质量后转化为付费用户
2. **推荐奖励**：用户推荐新用户，双方都获得免费 Skill 或折扣
3. **限时促销**：定期推出限时折扣，刺激购买
4. **企业推广**：向企业 PM 团队推广完整包，提供团队折扣

## 六、风险与应对

### 6.1 技术风险

**风险**：扣子平台的 Skill 开发环境与本地不同，可能出现兼容性问题

**应对**：
- 在扣子平台逐一测试所有技能
- 准备备用方案：如果扣子平台无法实现某些功能，可以提供"扣子平台版本"和"本地版本"两个版本

### 6.2 法律风险

**风险**：当前项目使用 CC BY-NC-SA 4.0 协议（非商业用途），如果上架扣子商店收费，可能违反协议

**应对**：
- **方案 A**：修改为商业友好的协议（如 MIT），但需要原作者的授权
- **方案 B**：保持非商业协议，只提供免费 Skill，通过企业咨询和培训变现
- **方案 C**：与原作者 Dean Peters 联系，获得商业使用的授权（推荐）

### 6.3 市场竞争风险

**风险**：扣子商店可能已经存在类似的 PM 技能，竞争激励

**应对**：
- **差异化定位**：强调"教学性"（ABC 原则），不仅生成交付物，还传授方法论
- **质量取胜**：参考原项目的"反模式"设计，提供更专业、更实用的技能
- **本土化优势**：针对中国 PM 的深度优化，是国外竞品无法比拟的

### 6.4 用户接受度风险

**风险**：PM 可能更习惯使用 Claude Code 或本地工具，不愿意切换到扣子平台

**应对**：
- **多平台覆盖**：不仅上架扣子商店，还保持 GitHub、Claude Code 插件市场的更新
- **云端优势宣传**：强调扣子的"无需安装、开箱即用、随时随地使用"优势
- **免费试用**：提供免费 Skill，降低用户尝试门槛

## 七、结论与建议

### 7.1 核心结论

1. **应该拆分，而非保持完整**：
   - 扣子平台的成功案例表明，"一个 Skill 解决一个具体问题"更符合用户习惯
   - 拆分后可以灵活定价，实现免费+付费混合模式
   - 拆分后维护成本可控（先按 Pack 拆分，未来可以按单个 Skill 拆分）

2. **定价策略应该是免费+付费混合模式**：
   - 基础 Skill 免费，降低用户尝试门槛
   - 高级 Skill 付费，实现变现
   - 完整包通过捐赠获取，符合开源精神

3. **需要完成云端化适配和深度中文本土化**：
   - 现有技能需要改造为云端输出（不依赖本地文件读写）
   - 中文本土化不能只是翻译，还要针对中国场景深度优化
